  <action>
    Create `packages/core/src/verbose-exec.ts`:

    - Module-level `let verbose = false`
    - `setVerbose(v: boolean)` — sets the flag, exported
    - `verboseExec(command: string, args: string[]): Promise<{ stdout: string; stderr: string }>` — wraps `promisify(execFile)`. Before calling, if verbose is true, logs `[prefix] command args...` to stderr. Prefix is colored: green `[adb]` for "adb" or "emulator" commands, blue `[simctl]` for "xcrun" commands. Use raw ANSI escape codes (not a library — consistent with existing pattern in log-viewer/manifest.ts). On error, log red `[error]` prefix with the failed command. Returns the same `{ stdout, stderr }` result.
    - `verboseSpawn(command: string, args: string[], opts?: SpawnOptions): ChildProcess` — wraps `spawn()`. Same verbose logging before spawning. Returns the ChildProcess.

    ANSI codes to use:
    ```
    const GREEN = "\x1b[32m";
    const BLUE = "\x1b[34m";
    const RED = "\x1b[31m";
    const DIM = "\x1b[2m";
    const RESET = "\x1b[0m";
    ```

    Log format when verbose: `${color}[prefix]${RESET} ${DIM}${command} ${args.join(" ")}${RESET}` to stderr via `process.stderr.write()` (not console.log — avoids polluting stdout which may be piped).

    For errors in verboseExec, catch the error, log it in red, then re-throw.

    Update `packages/core/src/index.ts` — add export:
    ```ts
    export { verboseExec, verboseSpawn, setVerbose } from "./verbose-exec.js";
    ```

    Update `packages/cli/src/index.ts`:
    - Add `import { setVerbose } from "@simvyn/core";` 
    - Add `.option("-v, --verbose", "Log every adb/simctl command before execution")` on the program (before registerStartCommand)
    - Add a preAction hook on the program to propagate the flag:
      ```ts
      program.hook("preAction", (thisCommand) => {
        const opts = thisCommand.optsWithGlobals();
        if (opts.verbose) setVerbose(true);
      });
      ```
    - This must be BEFORE `program.parse()` but AFTER the option is defined

    Update `packages/dashboard/vite.config.ts` — add to the `build` object:
    ```ts
    minify: false,
    sourcemap: true,
    ```
