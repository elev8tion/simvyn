  <action>
    Create `.github/workflows/release.yml` matching the react-native-duckdb pattern but adapted for simvyn's monorepo structure:

    ```yaml
    name: Release

    on:
      push:
        tags:
          - "v*"

    permissions:
      contents: write
      id-token: write

    jobs:
      publish:
        runs-on: ubuntu-latest
        environment: npm
        steps:
          - uses: actions/checkout@v4

          - uses: actions/setup-node@v4
            with:
              node-version: 24
              registry-url: https://registry.npmjs.org

          - name: Install dependencies
            run: npm ci

          - name: Build dashboard
            run: npm run build

          - name: Verify tarball contents
            working-directory: packages/cli
            run: npm pack --dry-run

          - name: Publish to npm
            working-directory: packages/cli
            run: npm publish --provenance --ignore-scripts
            env:
              NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

      github-release:
        needs: publish
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4

          - name: Create GitHub Release
            uses: softprops/action-gh-release@v2
            with:
              generate_release_notes: true
    ```

    Key differences from the reference:
    - Uses `npm ci` instead of `bun install` (this project uses npm workspaces, not bun)
    - Runs `npm run build` to build the dashboard (the CLI package serves built dashboard assets)
    - `npm publish` runs from `packages/cli/` directory (the publishable package in the monorepo)
    - No bun setup step needed

    The workflow requires:
    - A GitHub environment named "npm" with an `NPM_TOKEN` secret configured
    - The `id-token: write` permission enables npm provenance (OIDC-based supply chain attestation)

    Create the `.github/workflows/` directory if it doesn't exist.
