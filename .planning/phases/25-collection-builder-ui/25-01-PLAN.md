  <action>
Create the collections Zustand store at `packages/dashboard/src/panels/collections/stores/collections-store.ts` following the exact pattern of `deep-links-store.ts`:

**Store state:**
- `collections: Collection[]` — list of all collections (use Collection type from @simvyn/types)
- `actions: SerializedAction[]` — action catalog from GET /actions (define `SerializedAction` interface locally: `{ id: string; label: string; description: string; module: string; params: ActionParam[] }` — this is what the server returns after stripping execute/isSupported)
- `loading: boolean`
- `activeCollectionId: string | null` — which collection is being edited (null = list view)

**Store actions:**
- `fetchCollections()` — GET /api/modules/collections/, set collections
- `fetchActions()` — GET /api/modules/collections/actions, set actions
- `createCollection(name: string)` — POST /api/modules/collections/ with `{ name }`, add to collections array, set activeCollectionId to new id, toast success
- `updateCollection(id: string, data: { name?: string; description?: string; steps?: CollectionStep[] })` — PUT /api/modules/collections/:id, update in array, toast success
- `deleteCollection(id: string)` — DELETE /api/modules/collections/:id, remove from array, clear activeCollectionId if it was active, toast success
- `duplicateCollection(id: string)` — POST /api/modules/collections/:id/duplicate, add result to collections, toast success
- `setActiveCollectionId(id: string | null)` — set activeCollectionId

Use `toast` from sonner for success/error feedback. Import ActionParam and CollectionStep from @simvyn/types.

**Module registration (3 files):**

1. `module-store.ts` — Add `"collections"` to DOCK_ORDER array, positioned BEFORE `"device-settings"` (after `"crash-logs"`).

2. `module-icons.tsx` — Add a `CollectionsIcon` component with the same liquid glass SVG pattern as existing icons. Use accent color `#E879F9` (fuchsia/purple — distinct from all existing icons). The icon should represent a stacked layers concept (3 overlapping rounded rectangles stacked vertically with slight offset, like a playlist/stack). Add to `moduleIconMap` as `"collections": CollectionsIcon` and to `moduleLabelMap` as `"collections": "Collections"`.

3. `App.tsx` — Add `import "./panels/CollectionsPanel";` as a side-effect import alongside the existing panel imports (after ToolSettingsPanel).

Create `packages/dashboard/src/panels/CollectionsPanel.tsx` following the pattern of DeepLinksPanel.tsx.

**Panel structure:** The panel has two views controlled by `activeCollectionId`:
- When `null` → show collection list view (this task)
- When set → show step builder view (Plan 02 will add this; for now render a placeholder `<div>` with the collection name and a "Back" button that sets activeCollectionId to null)

**List view implementation:**

1. **Header:** "Collections" title (text-base font-medium text-text-primary) with a "New Collection" glass-button-primary on the right. Clicking "New Collection" prompts for a name — use a small inline form (glass-input + submit button) that toggles visible, NOT window.prompt. On submit, call `createCollection(name)`.

2. **Empty state:** When no collections exist, show `<div className="glass-panel"><p className="glass-empty-state">No collections yet — create one to get started</p></div>`.

3. **Collection cards:** Map over `collections` array. Each card is a `<div className="glass-panel px-4 py-3 cursor-pointer hover:bg-bg-surface/30 transition-colors">` containing:
   - Collection name (text-sm font-medium text-text-primary)
   - Step count badge: `"{n} steps"` in a small muted span (text-[10px] text-text-muted)
   - Description if present (text-xs text-text-secondary truncate)
   - Updated timestamp: `"Updated {relative time}"` (text-[10px] text-text-muted) — use simple relative time like "just now", "2h ago", etc.
   - Action buttons (on hover, group-hover:opacity-100 opacity-0 transition): Duplicate (Copy icon) and Delete (Trash2 icon from lucide-react). Delete calls `deleteCollection(id)`, Duplicate calls `duplicateCollection(id)`.
   - Clicking the card body (not action buttons) sets `setActiveCollectionId(id)` to enter the builder.

4. **useEffect on mount:** Call `fetchCollections()` and `fetchActions()` once on mount.

5. **Register panel:** At bottom of file: `registerPanel("collections", CollectionsPanel)` and `export default CollectionsPanel`.

Use lucide-react icons: Plus, Copy, Trash2, ArrowLeft, Layers (for the header icon if desired). All glass- utility classes from main.css.
