**StepBuilder.tsx** — The main step builder view using Framer Motion's Reorder for drag-and-drop.

Props: `{ collectionId: string; onBack: () => void }`

Implementation:
1. Read collection from store: `useCollectionsStore(s => s.collections.find(c => c.id === collectionId))`. Read `actions` and `updateCollection` from store.

2. **Local step state:** Use `useState<CollectionStep[]>` initialized from collection.steps. This local state enables drag reorder without immediate server roundtrips. Sync back to server on meaningful changes (add step, remove step, reorder, param change) via a debounced or explicit save.

3. **Header:** Back arrow button (ArrowLeft icon, glass-button, calls `onBack`), editable collection name (inline text input, glass-input, on blur or Enter save via `updateCollection(id, { name })`), and a step count label (text-text-muted).

4. **"Add Step" button:** glass-button-primary with Plus icon. Clicking toggles `showActionPicker` state. Render `<ActionPicker>` when open. On add: create a new CollectionStep `{ id: crypto.randomUUID(), actionId, params: {} }` with default param values pre-filled from the action descriptor's defaultValue fields. Append to local steps array. Save to server via `updateCollection(id, { steps: newSteps })`.

5. **Step list with drag-and-drop (CBLD-03):** Use Framer Motion's `<Reorder.Group>` and `<Reorder.Item>`:
   ```tsx
   import { Reorder } from "framer-motion";

   <Reorder.Group axis="y" values={steps} onReorder={handleReorder} className="space-y-2">
     {steps.map((step, i) => (
       <Reorder.Item key={step.id} value={step}>
         <StepCard
           step={step}
           action={actions.find(a => a.id === step.actionId)}
           index={i}
           onRemove={() => handleRemoveStep(step.id)}
           onUpdateParams={(params) => handleUpdateParams(step.id, params)}
         />
       </Reorder.Item>
     ))}
   </Reorder.Group>
   ```

6. **handleReorder:** `(newOrder: CollectionStep[]) => { setSteps(newOrder); updateCollection(id, { steps: newOrder }); }` — saves immediately on reorder.

7. **handleRemoveStep:** Filter out the step, update local state, save to server.

8. **handleUpdateParams:** Map over steps, update the matching step's params, save to server. To avoid excessive API calls on every keystroke, debounce the save: use a ref-based approach — update local state immediately, but defer the PUT call by 500ms (clear previous timeout on new change). Import no new libraries for this; use setTimeout/clearTimeout with useRef.

9. **Empty state:** When steps array is empty, show `<div className="glass-panel"><p className="glass-empty-state">No steps yet — click "Add Step" to browse available actions</p></div>`.
