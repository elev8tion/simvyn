  <name>Task 1: Add devicectl helpers, physical device discovery, and devicectl-backed methods</name>
  <files>packages/core/src/adapters/ios.ts</files>
  <action>
Add the following to the iOS adapter file (packages/core/src/adapters/ios.ts):

1. **`devicectlJson<T>(args: string[]): Promise<T>` helper function** (module-level, before createIosAdapter):
   - Create a temp directory via `mkdtemp(join(tmpdir(), "simvyn-dctl-"))`
   - Set `jsonPath = join(tmpDir, "output.json")`
   - Run `verboseExec("xcrun", ["devicectl", ...args, "--json-output", jsonPath, "-q"])`
   - Read and parse the JSON file: `import { readFile } from "node:fs/promises"` (already imported)
   - Return `parsed.result as T`
   - Always clean up the temp dir in a `finally` block with `rm(tmpDir, { recursive: true, force: true })`

2. **`isPhysicalDevice(id: string): boolean` helper** (module-level, exported):
   - Returns `id.startsWith("physical:")`

3. **`stripPhysicalPrefix(id: string): string` helper** (module-level, exported):
   - Returns `id.replace("physical:", "")`

4. **`hasDevicectl` variable** (module-level, `let hasDevicectl: boolean | null = null`):
   - Lazy-checked on first call via a `checkDevicectl()` async function that runs `verboseExec("xcrun", ["devicectl", "list", "devices", "--help"])` wrapped in try/catch. Cache the result.

5. **`DevicectlListResult` interface** matching the research JSON structure:
   ```typescript
   interface DevicectlDevice {
     identifier: string;
     hardwareProperties: {
       reality: string;
       marketingName: string;
       platform: string;
       udid: string;
       deviceType: string;
     };
     deviceProperties: {
       name: string;
       osVersionNumber: string;
       ddiServicesAvailable?: boolean;
     };
     connectionProperties: {
       transportType: string;
       tunnelState: string;
       pairingState: string;
     };
   }
   interface DevicectlListResult { devices: DevicectlDevice[]; }
   ```

6. **`listPhysicalDevices(): Promise<Device[]>` function** (module-level):
   - If `hasDevicectl` is false, return `[]`
   - Call `devicectlJson<DevicectlListResult>(["list", "devices"])`
   - Filter: `d.hardwareProperties.reality === "physical"` AND `d.hardwareProperties.platform === "iOS"`
   - Map to Device objects:
     - `id: "physical:" + d.identifier`
     - `name: d.deviceProperties.name`
     - `platform: "ios"`
     - `state: "booted"` (physical devices are always "on" if visible)
     - `osVersion: "iOS " + d.deviceProperties.osVersionNumber`
     - `deviceType: d.hardwareProperties.marketingName` (e.g., "iPhone 15 Pro")
     - `isAvailable: true`
   - Wrap in try/catch — on error, log warning and return `[]`

7. **Update `listDevices()`** in the adapter:
   - After getting simulator devices, call `await listPhysicalDevices()` and concat the results
   - Check devicectl availability lazily on first call (call `checkDevicectl()` if `hasDevicectl === null`)

8. **Add physical device branches to methods that USE devicectl** — for each method, check `isPhysicalDevice(deviceId)` at the top. If physical, route to devicectl. If not, use existing simctl path unchanged. These are the methods with real devicectl implementations:
   - `boot(id)` — if physical, no-op (physical devices don't need booting)
   - `shutdown(id)` — if physical, throw `"Shutdown is not available for physical devices"`
   - `listApps(deviceId)` — if physical, use devicectl: `devicectlJson(["device", "info", "apps", "--device", stripPhysicalPrefix(deviceId)])`. Map result to AppInfo[] (bundleIdentifier→bundleId, bundleDisplayName→name, bundleShortVersion→version, type: "user").
   - `installApp(deviceId, appPath)` — if physical, use `verboseExec("xcrun", ["devicectl", "device", "install", "app", "--device", stripPhysicalPrefix(deviceId), appPath, "-q"])`. No IPA unzip needed — devicectl handles .ipa directly.
   - `uninstallApp(deviceId, bundleId)` — if physical, use `verboseExec("xcrun", ["devicectl", "device", "uninstall", "app", "--device", stripPhysicalPrefix(deviceId), bundleId, "-q"])`
   - `launchApp(deviceId, bundleId)` — if physical, use `verboseExec("xcrun", ["devicectl", "device", "process", "launch", "--device", stripPhysicalPrefix(deviceId), bundleId, "-q"])`
   - `terminateApp(deviceId, bundleId)` — if physical, implement two-step: (1) list processes via `devicectlJson(["device", "info", "processes", "--device", stripPhysicalPrefix(deviceId)])`, (2) find process by executable name matching bundleId or similar, (3) terminate via `verboseExec("xcrun", ["devicectl", "device", "process", "terminate", "--device", stripPhysicalPrefix(deviceId), "--pid", String(pid), "-q"])`. If process not found, silently succeed (app may already be terminated).
   - `getAppInfo` — if physical, call listApps and filter by bundleId
   - `openUrl(deviceId, url)` — if physical, use `verboseExec("xcrun", ["devicectl", "device", "process", "launch", "--device", stripPhysicalPrefix(deviceId), "com.apple.mobilesafari", "--payload-url", url, "-q"])`
   - `collectBugReport` — if physical, use devicectl sysdiagnose: `verboseExec("xcrun", ["devicectl", "device", "sysdiagnose", "--device", stripPhysicalPrefix(deviceId), "-q"])` and handle the output path accordingly. If this is complex, throw unsupported instead.

Do NOT create a separate adapter class — all branching happens inside the single adapter per the locked user decision.
