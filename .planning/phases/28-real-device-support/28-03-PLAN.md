   ```typescript
   const device = fastify.deviceManager.devices.find((d: Device) => d.id === deviceId);

   const adapter = fastify.deviceManager.getAdapter(device.platform);
   ```

3. Update each capability flag to account for physical device restrictions. Use the research feature matrices:

   **iOS physical device — unavailable features:**
   - appearance (no `simctl ui` equivalent in devicectl)
   - statusBar (simctl-only)
   - permissions (simctl privacy — no devicectl equivalent)
   - locale (simctl spawn defaults — no devicectl equivalent)
   - contentSize (simctl ui — no devicectl equivalent)
   - increaseContrast (simctl ui — no devicectl equivalent)
   - bugReport — KEEP as available (devicectl sysdiagnose works)

   **Android physical device — unavailable features:**
   - locale (requires root)
   - batterySimulation — KEEP (dumpsys battery set works on physical, just resets on reboot)
   - Everything else works via adb shell

   Updated return object:
   ```typescript
   const isIosPhysical = isPhysical && device.platform === "ios";
   const isAndroidPhys = isPhysical && device.platform === "android";

   return {
     appearance: !!adapter?.setAppearance && !isIosPhysical,
     statusBar: !!adapter?.setStatusBar && !isPhysical,
     permissions: !!adapter?.grantPermission && !isIosPhysical,
     resetPermissions: !!adapter?.resetPermissions && !isIosPhysical,
     locale: !!adapter?.setLocale && !isPhysical,
     contentSize: !!adapter?.setContentSize && !isIosPhysical,
     increaseContrast: !!adapter?.setIncreaseContrast && !isIosPhysical,
     talkBack: !!adapter?.setTalkBack,  // Android only, works on physical
     portForward: !!adapter?.addForward,  // Works on physical Android
     displayOverride: !!adapter?.setDisplaySize,  // Works on physical Android
     batterySimulation: !!adapter?.setBattery,  // Works on physical Android (simulation resets on reboot)
     inputInjection: !!adapter?.inputTap,  // Works on physical Android
     bugReport: !!adapter?.collectBugReport,  // Works on both (sysdiagnose on iOS, bugreport on Android)
   };
   ```

Note: `statusBar` uses `!isPhysical` (neither platform supports it on real devices). `locale` uses `!isPhysical` (iOS: no simctl spawn on device; Android: requires root). Other iOS-specific features use `!isIosPhysical`.

**Part A: DeviceManager disconnect detection** (`packages/core/src/device-manager.ts`):

1. Update the `DeviceManager` interface to add:
   ```typescript
   on(event: "devices-disconnected", cb: (devices: Device[]) => void): void;
   off(event: "devices-disconnected", cb: (devices: Device[]) => void): void;
   ```
   (Add as overloads alongside the existing "devices-changed" event signatures.)

2. In the `poll()` function, after computing `allDevices` and detecting a fingerprint change, detect disconnected physical devices:
   ```typescript
   if (fp !== lastFingerprint) {
     const currentIds = new Set(allDevices.map(d => d.id));
     const disconnected = currentDevices.filter(
       d => d.deviceType === "Physical" && !currentIds.has(d.id)
     );

     lastFingerprint = fp;
     currentDevices = allDevices;
     emitter.emit("devices-changed", allDevices);

     if (disconnected.length > 0) {
       emitter.emit("devices-disconnected", disconnected);
     }
   }
   ```

   The order matters: update `currentDevices` before emitting `devices-disconnected` so listeners see the new device list when they check.

**Part B: Collections action registry device-aware isSupported** (`packages/modules/collections/action-registry.ts`):

The current `isSupported` signature is `(adapter: PlatformAdapter) => boolean`. To make it device-aware without changing the type, the execution engine needs to add device-type checking at execution time.

Update the execution engine (`packages/modules/collections/execution-engine.ts`) to add a physical device guard BEFORE calling `action.isSupported`:

In the `Promise.all(devices.map(...))` loop, after getting the adapter, add:
```typescript
// Check if this action uses features unavailable on physical devices
const isPhysDevice = device.id.startsWith("physical:") ||
  (!device.id.startsWith("emulator-") && !device.id.startsWith("avd:") && device.platform === "android");
```

Then use a Set of action IDs that are unsupported on physical devices:
```typescript
const PHYSICAL_UNSUPPORTED_IOS = new Set([
  "set-location", "clear-location", "set-status-bar", "clear-status-bar",
  "set-clipboard", "set-appearance", "set-content-size", "set-increase-contrast",
  "erase-device",
]);
const PHYSICAL_UNSUPPORTED_ANDROID = new Set([
  "set-location", "clear-location", "set-locale", "erase-device",
]);
```

In the execution loop, before the existing `!action.isSupported(adapter)` check, add:
```typescript
if (isPhysDevice) {
  const unsupported = device.platform === "ios" ? PHYSICAL_UNSUPPORTED_IOS : PHYSICAL_UNSUPPORTED_ANDROID;
  if (unsupported.has(collectionStep.actionId)) {
    dr.status = "skipped";
    dr.completedAt = new Date().toISOString();
    return;
  }
}
```

Define these sets at module level (outside the function) for efficiency. The `device` parameter in the `devices.map()` callback has `platform` from the `RunCollectionOpts.devices` array — verify this field is available. If the device object in the execution engine doesn't carry platform, look it up from the adapter's platform field.

This approach avoids changing the ActionDescriptor type signature while properly skipping physical-device-incompatible steps.

Per the user decision: "File system / database browsing: mark as unsupported on real iOS devices (no filesystem access without jailbreak). Android real devices work via adb as normal."

**Part A: File-system route guard** (`packages/modules/file-system/routes.ts`):

1. Import `isPhysicalDevice` from `@simvyn/core`.

2. Add a guard at the top of every route handler that receives a `deviceId` parameter. Before any file operation, check:
   ```typescript
   if (isPhysicalDevice(deviceId)) {
     return reply.status(400).send({
       error: "File system browsing is not available on physical iOS devices (no filesystem access without jailbreak)"
     });
   }
   ```

   The simplest approach: add a Fastify `preHandler` hook or add the guard to the plugin's `onRequest` hook so it applies to ALL routes in this module at once. Alternatively, add the check at the top of each route handler that extracts `deviceId` from params or query.

   Note: Android physical devices do NOT need this guard — `adb shell run-as` works on physical Android devices. The guard should only trigger for iOS physical devices: `if (isPhysicalDevice(deviceId))`. Since only iOS devices have the `physical:` prefix, this naturally only blocks iOS physical devices.

**Part B: Database route guard** (`packages/modules/database/routes.ts`):

1. Import `isPhysicalDevice` from `@simvyn/core`.

2. Same pattern as file-system: add a guard for iOS physical device IDs:
   ```typescript
   if (isPhysicalDevice(deviceId)) {
     return reply.status(400).send({
       error: "Database browsing is not available on physical iOS devices (no filesystem access without jailbreak)"
     });
   }
   ```

   Apply to all database routes that accept a deviceId parameter (list databases, query, prefs read/write, etc.).
