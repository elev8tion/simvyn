<task type="auto">
  <name>Task 1: Add isAndroidPhysical helper and guard emulator-only methods</name>
  <files>packages/core/src/adapters/android.ts</files>
  <action>
1. **Add `isAndroidPhysical(deviceId: string): boolean` helper** (module-level, exported):
   - Returns `!deviceId.startsWith("emulator-") && !deviceId.startsWith("avd:")`
   - This correctly identifies physical device serials (e.g., `R5CR30XXXXX`, `192.168.1.5:5555`) as distinct from emulators (`emulator-5554`) and shutdown AVDs (`avd:Pixel_6`)

2. **Add `isEmulatorRunning(deviceId: string): boolean` helper** (module-level):
   - Returns `deviceId.startsWith("emulator-")`

3. **Add physical device guards to these emulator-only methods:**

   - **`setLocation(deviceId, lat, lon)`** — currently has no guard. Add at top:
     ```typescript
     if (deviceId.startsWith("avd:")) throw new Error("Device must be booted");
     if (isAndroidPhysical(deviceId)) {
       throw new Error("Location simulation is not available on physical Android devices");
     }
     ```

   - **`clearLocation(deviceId)`** — same guard pattern as setLocation:
     ```typescript
     if (deviceId.startsWith("avd:")) throw new Error("Device must be booted");
     if (isAndroidPhysical(deviceId)) {
       throw new Error("Location simulation is not available on physical Android devices");
     }
     ```

   - **`shutdown(id)`** — currently catches errors silently for physical devices (the `adb emu kill` will fail but is caught). Update the comment to be explicit and add a physical device early return:
     ```typescript
     if (id.startsWith("avd:")) return; // can't shut down something not running
     if (isAndroidPhysical(id)) return; // no-op for physical devices — don't attempt emu kill
     ```
     This prevents even trying `adb emu kill` on physical devices. The existing try/catch already handles this, but the explicit guard makes intent clear and avoids the exec overhead.

   - **`setLocale(deviceId, locale)`** — currently uses `adb shell su 0 setprop` which requires root. Physical devices are almost never rooted. Add guard:
     ```typescript
     if (isAndroidPhysical(deviceId)) {
       throw new Error("Locale change requires root access and is not available on most physical devices");
     }
     ```
     (The avd: guard already exists if present; if not, add it too.)

4. **Export `isAndroidPhysical`** — add to `packages/core/src/index.ts` barrel:
   ```typescript
   export { isAndroidPhysical } from "./adapters/android.js";
   ```

5. **Do NOT add guards to adb shell methods** — these all work identically on physical and emulator devices: listApps, installApp, uninstallApp, launchApp, terminateApp, screenshot, startRecording, stopRecording, setClipboard, setAppearance, addMedia, grantPermission, etc. They should continue to work without any physical device check.
