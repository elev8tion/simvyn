  <action>
1. Install cmdk: `npm install cmdk -w @simvyn/dashboard`

2. Create `packages/dashboard/src/components/CommandPalette.tsx`:
   - Use `cmdk` package's `Command` component (headless, unstyled)
   - Component manages open/closed state with `useState<boolean>`
   - Register global `keydown` listener for Cmd+K / Ctrl+K to open
   - When open, render a **dialog overlay** (fixed position, full-screen backdrop with `rgba(0,0,0,0.5)` + `backdrop-filter: blur(8px)`) and a centered command palette container
   - Use `<Command>` → `<Command.Input>` → `<Command.List>` → `<Command.Group>` → `<Command.Item>` structure
   
   **Module navigation group ("Modules"):**
   - Import `moduleIconMap` and `moduleLabelMap` from `../components/icons/module-icons`
   - Import `useModuleStore` to get `modules` list (for dynamic module availability)
   - Import `useNavigate` from `react-router`
   - For each available module, render a `<Command.Item>` with:
     - The module's custom icon from `moduleIconMap` (size={18})
     - The module's label from `moduleLabelMap`
     - The module's description from `modules[].description`
     - `onSelect` → `navigate('/${mod.name}')` then close palette
   
   **Device actions group ("Actions"):**
   - Hardcode a list of device action items:
     - "Take Screenshot" — calls `POST /api/modules/screenshot/capture` for first selected device
     - "Toggle Dark Mode" — calls `POST /api/modules/settings/appearance` with toggle
     - "Set Location" — navigates to `/location`
     - "Clear Logs" — navigates to `/logs`
     - "Erase Device" — navigates to `/devices`
   - For actions that call APIs: use `fetch()`, show sonner toast on success/error, close palette
   - For actions that navigate: use `navigate()`, close palette
   - Each action item shows a small icon (can use simple SVG or text emoji) + label + description
   
   **Escape/outside click to close:**
   - cmdk's `<Command.Dialog>` handles this — OR use a manual approach with the overlay div's onClick
   - Also close on Escape key (register in the keydown handler)
   
   **Empty state:**
   - `<Command.Empty>` → "No results found"

3. Update `App.tsx`:
   - Import `CommandPalette` 
   - Render `<CommandPalette />` inside `<WsProvider>` but outside `<Routes>` (it's a global overlay, not route-dependent)
   - Place it after the Toaster component

4. Add Liquid Glass CSS in `main.css`:
   Add styles for the command palette classes. Use class names like:
   - `.cmdk-overlay` — fixed inset-0, z-50, backdrop blur
   - `.cmdk-dialog` — centered glass panel (wider than normal, e.g., max-width 560px, max-height 400px), frosted glass bg, rounded corners, border, shadow
   - `.cmdk-input` — transparent background, white text, no outline, padding, border-bottom
   - `.cmdk-list` — scrollable, padding
   - `.cmdk-group-heading` — uppercase, muted text, small font, padding
   - `.cmdk-item` — flex, gap, padding, rounded, hover state with subtle glass highlight
   - `.cmdk-item[data-selected="true"]` or `.cmdk-item.selected` — active/selected item highlight (similar to dock-icon active bg)
   - `.cmdk-empty` — centered muted text
   
   Use the same glass values as the rest of the dashboard:
   - Background: `rgba(30, 30, 45, 0.85)` (slightly more opaque for readability)
   - Backdrop: `blur(24px) saturate(1.4)`
   - Border: `1px solid rgba(255, 255, 255, 0.1)`
   - Shadow: deep box-shadow matching glass-panel pattern
   - Input: transparent bg, `var(--color-text-primary)` color
   - Selected item: `rgba(255, 255, 255, 0.08)` background

NOTE about cmdk: The library uses `[cmdk-*]` data attributes for styling. The CSS selectors should use these attribute selectors, e.g., `[cmdk-input]`, `[cmdk-item]`, `[cmdk-group-heading]`, `[cmdk-list]`, `[cmdk-separator]`. Check the cmdk docs/source for the exact attribute names. Alternatively, apply className props to each cmdk element and style via those classes.
