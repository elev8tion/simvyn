  <action>
Create `packages/dashboard/src/stores/favourite-store.ts` as a Zustand store:

```
interface FavouriteStore {
  favouriteIds: Set<string>;
  loaded: boolean;
  load: () => Promise<void>;
  toggle: (id: string) => Promise<void>;
  isFavourite: (id: string) => boolean;
}
```

- `load()` — `GET /api/modules/devices/favourites`, populate `favouriteIds` Set, set `loaded: true`
- `toggle(id)` — if favourite, `DELETE /api/modules/devices/favourites` with `{ deviceId: id }`, remove from local Set. If not favourite, `POST /api/modules/devices/favourites` with `{ deviceId: id }`, add to local Set. Optimistic update: mutate the Set first, revert on fetch error.
- `isFavourite(id)` — return `favouriteIds.has(id)`

Use `Set<string>` internally but convert to/from array for serialization. Call `load()` on store creation (immediate invocation in the store definition, same pattern as other stores that auto-fetch).
