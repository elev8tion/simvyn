# Phase 28.1: Favourite Devices - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Persist a set of favourite device IDs so they appear at the top of the DeviceSelector dropdown. Users can toggle favourites via star icon or right-click context menu. Includes CLI commands for managing favourites. This enhances the existing DeviceSelector — no new modules or panels.

</domain>

<decisions>
## Implementation Decisions

### Favouriting interaction
- Star icon always visible on every device row — filled star for favourites, outline star for non-favourites
- Click star to toggle favourite on/off immediately — no confirmation dialog
- Right-click context menu on device rows as alternative — "Add to Favourites" / "Remove from Favourites"
- CLI commands: `simvyn device favourite <id>`, `simvyn device unfavourite <id>`, `simvyn device favourites` (list)

### Selector ordering
- Dedicated "Favourites" section at the top of the DeviceSelector dropdown, above Physical > Simulators > Emulators groups
- Favourite devices move out of their original group into the Favourites section — no duplicate entries
- Within the Favourites section, sub-headers for platform grouping: "iOS" and "Android" sub-groups
- Favourites section header always visible, even when empty — show a subtle hint like "Star devices to add them here"

### Scope & persistence
- One flat list of favourite device IDs stored at `~/.simvyn/favourites.json` — core storage, not module-specific
- Silent cleanup of stale device IDs — remove favourites for devices that no longer exist, automatically on startup/poll
- Stale cleanup should account for temporarily disconnected physical devices (USB unplug) — only clean up IDs not seen across multiple polls, not on first absence

### Claude's Discretion
- Order of favourites within the section (chronological, alphabetical, or other)
- Exact star icon style and size — should fit the Liquid Glass aesthetic
- Right-click context menu implementation approach
- Cleanup timing and threshold for stale device detection

</decisions>

<specifics>
## Specific Ideas

- User has ~30 simulators (iPhones, Watches, iPads) but regularly uses only 3 — favourites solve this by surfacing the 3 used devices at the top
- The DeviceSelector already groups by type (Physical > Simulators > Emulators) from Phase 28 — Favourites adds a new top-level group above these

</specifics>
