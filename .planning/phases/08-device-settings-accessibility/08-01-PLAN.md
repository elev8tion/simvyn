   - Add these optional methods to `PlatformAdapter`:

     ```
     setAppearance?(deviceId: string, mode: "light" | "dark"): Promise<void>;
     setStatusBar?(deviceId: string, overrides: Record<string, string>): Promise<void>;
     clearStatusBar?(deviceId: string): Promise<void>;
     grantPermission?(deviceId: string, bundleId: string, permission: string): Promise<void>;
     revokePermission?(deviceId: string, bundleId: string, permission: string): Promise<void>;
     resetPermissions?(deviceId: string, bundleId: string): Promise<void>;
     setLocale?(deviceId: string, locale: string): Promise<void>;
     setContentSize?(deviceId: string, size: string): Promise<void>;
     setIncreaseContrast?(deviceId: string, enabled: boolean): Promise<void>;
     setTalkBack?(deviceId: string, enabled: boolean): Promise<void>;
     ```

2. In `packages/core/src/adapters/ios.ts`:
   - Add `setAppearance`: `xcrun simctl ui <deviceId> appearance <light|dark>`
   - Add `setStatusBar`: `xcrun simctl status_bar <deviceId> override` with flags for each override key. Supported keys: `--time <string>`, `--batteryLevel <0-100>`, `--batteryState <charged|charging|discharging>`, `--cellularBars <0-4>`, `--wifiBars <0-3>`, `--operatorName <string>`, `--dataNetwork <wifi|3g|4g|lte|lte-a|lte+|5g|5g+|5g-uwb>`. Map Record keys to these flags.
   - Add `clearStatusBar`: `xcrun simctl status_bar <deviceId> clear`
   - Add `grantPermission`: `xcrun simctl privacy <deviceId> grant <permission> <bundleId>`. Common permissions: photos, camera, microphone, contacts, calendar, reminders, location, motion, health, homekit, siri, speech, media-library, bluetooth-peripheral.
   - Add `revokePermission`: `xcrun simctl privacy <deviceId> revoke <permission> <bundleId>`
   - Add `resetPermissions`: `xcrun simctl privacy <deviceId> reset all <bundleId>`
   - Add `setLocale`: `xcrun simctl spawn <deviceId> defaults write "Apple Global Domain" AppleLocale -string <locale>` then `xcrun simctl spawn <deviceId> defaults write "Apple Global Domain" AppleLanguages -array <langCode>` (extract lang code from locale like "en" from "en_US"). Note: locale change requires a reboot to take effect — log this but don't auto-reboot.
   - Add `setContentSize`: `xcrun simctl ui <deviceId> content_size <size>`. Valid sizes: extra-small, small, medium (default), large, extra-large, extra-extra-large, extra-extra-extra-large, accessibility-medium, accessibility-large, accessibility-extra-large, accessibility-extra-extra-large, accessibility-extra-extra-extra-large.
   - Add `setIncreaseContrast`: `xcrun simctl ui <deviceId> increase_contrast <enabled|disabled>`
   - Add `"settings"` and `"accessibility"` to the `capabilities()` return array
   - Set `setTalkBack: undefined` (iOS does not have TalkBack)

3. In `packages/core/src/adapters/android.ts`:
   - Add `setAppearance`: `adb -s <deviceId> shell cmd uimode night <yes|no>` (map "dark" to "yes", "light" to "no")
   - Add `grantPermission`: `adb -s <deviceId> shell pm grant <bundleId> android.permission.<PERMISSION>`. The permission param comes in as a short name (e.g., "CAMERA") — prepend `android.permission.` prefix.
   - Add `revokePermission`: `adb -s <deviceId> shell pm revoke <bundleId> android.permission.<PERMISSION>`. Same prefix logic.
   - Add `setLocale`: `adb -s <deviceId> shell setprop persist.sys.locale <locale>` then `adb -s <deviceId> shell setprop ctl.restart zygote` to apply. Warn: this restarts the runtime.
   - Add `setTalkBack`: 
     - Enable: `adb -s <deviceId> shell settings put secure enabled_accessibility_services com.google.android.marvin.talkback/com.google.android.marvin.talkback.TalkBackService`
     - Disable: `adb -s <deviceId> shell settings put secure enabled_accessibility_services ""`
     - Both: also set `adb -s <deviceId> shell settings put secure accessibility_enabled <1|0>`
   - Add `"settings"` and `"accessibility"` to the `capabilities()` return array
   - Set `setStatusBar: undefined`, `clearStatusBar: undefined` (Android has no equivalent)
   - Set `resetPermissions: undefined` (Android has no reset-all equivalent — only individual grant/revoke)
   - Set `setContentSize: undefined`, `setIncreaseContrast: undefined` (no equivalent adb commands)
   - Guard all new methods against `avd:` prefix device IDs with the same pattern used by existing methods

4. Create `packages/modules/settings/routes.ts`:
   Export `async function settingsRoutes(fastify: FastifyInstance)` with these endpoints:

   **POST /appearance** — Set dark/light mode
   - Body: `{ deviceId: string, mode: "light" | "dark" }`
   - Calls `adapter.setAppearance(device.id, mode)`
   - Returns `{ success: true }`

   **POST /status-bar** — Override iOS status bar
   - Body: `{ deviceId: string, overrides: Record<string, string> }`
   - Calls `adapter.setStatusBar(device.id, overrides)`
   - Returns `{ success: true }`
   - Return 400 if adapter doesn't support setStatusBar

   **POST /status-bar/clear** — Clear status bar overrides
   - Body: `{ deviceId: string }`
   - Calls `adapter.clearStatusBar(device.id)`

   **POST /permission/grant** — Grant app permission
   - Body: `{ deviceId: string, bundleId: string, permission: string }`
   - Calls `adapter.grantPermission(device.id, bundleId, permission)`

   **POST /permission/revoke** — Revoke app permission
   - Body: `{ deviceId: string, bundleId: string, permission: string }`
   - Calls `adapter.revokePermission(device.id, bundleId, permission)`

   **POST /permission/reset** — Reset all permissions for app
   - Body: `{ deviceId: string, bundleId: string }`
   - Calls `adapter.resetPermissions(device.id, bundleId)`
   - Return 400 if adapter doesn't support resetPermissions

   **POST /locale** — Change locale
   - Body: `{ deviceId: string, locale: string }`
   - Calls `adapter.setLocale(device.id, locale)`

   **POST /content-size** — Set accessibility content size
   - Body: `{ deviceId: string, size: string }`
   - Calls `adapter.setContentSize(device.id, size)`
   - Return 400 if not supported

   **POST /increase-contrast** — Toggle increase contrast
   - Body: `{ deviceId: string, enabled: boolean }`
   - Calls `adapter.setIncreaseContrast(device.id, enabled)`
   - Return 400 if not supported

   **POST /talkback** — Toggle TalkBack
   - Body: `{ deviceId: string, enabled: boolean }`
   - Calls `adapter.setTalkBack(device.id, enabled)`
   - Return 400 if not supported

   **GET /capabilities** — Get settings capabilities for a device
   - Query: `?deviceId=<id>`
   - Return object with boolean flags for each feature based on platform:
     ```
     {
       appearance: true,      // both platforms
       statusBar: boolean,    // iOS only
       permissions: true,     // both (grant/revoke)
       resetPermissions: boolean, // iOS only
       locale: true,          // both
       contentSize: boolean,  // iOS only
       increaseContrast: boolean, // iOS only
       talkBack: boolean,     // Android only
     }
     ```
   - Derive from adapter method presence (e.g., `!!adapter?.setStatusBar`)

   Follow the same error handling pattern as deep-links/routes.ts:
   - 404 for device not found
   - 400 for device not booted
   - 400 for unsupported capability
   - try/catch with 500 + error message

  <action>
Create `packages/modules/settings/cli.ts` exporting two functions:

1. `registerSettingsCli(program: Command)`:
   Create a `settings` command group with subcommands:

   **`simvyn settings dark-mode <device> <on|off>`**
   - Maps "on" to "dark", "off" to "light"
   - Calls adapter.setAppearance()
   - Prints "Dark mode enabled/disabled on <device.name>"

   **`simvyn settings status-bar <device>`**
   - Options: `--time <string>`, `--battery-level <0-100>`, `--battery-state <charged|charging|discharging>`, `--cellular-bars <0-4>`, `--wifi-bars <0-3>`, `--operator <string>`, `--network <type>`, `--clear` (boolean flag)
   - If --clear, call adapter.clearStatusBar()
   - Otherwise build overrides Record from provided options (only include non-undefined), call adapter.setStatusBar()
   - Prints "Status bar updated on <device.name>" or "Status bar cleared on <device.name>"

   **`simvyn settings permission <device> <grant|revoke|reset> <permission> [bundleId]`**
   - grant/revoke require both permission and bundleId arguments
   - reset requires bundleId but not permission (resets all)
   - Calls corresponding adapter method
   - Prints confirmation

   **`simvyn settings locale <device> <locale>`**
   - Calls adapter.setLocale()
   - Prints "Locale set to <locale> on <device.name> (reboot may be required)"

2. `registerA11yCli(program: Command)`:
   Create an `a11y` command group with subcommands:

   **`simvyn a11y content-size <device> <size>`**
   - size is one of the valid content size strings
   - Calls adapter.setContentSize()
   - Prints "Content size set to <size>"

   **`simvyn a11y increase-contrast <device> <on|off>`**
   - Maps on/off to boolean
   - Calls adapter.setIncreaseContrast()
   - Prints "Increase contrast enabled/disabled"

   **`simvyn a11y talkback <device> <on|off>`**
   - Calls adapter.setTalkBack()
   - Prints "TalkBack enabled/disabled"

Follow the same CLI pattern as other modules:
- Lazy-import `@simvyn/core` inside action handlers
- Create adapters + device manager fresh each invocation
- Match devices by full ID or prefix
- Always call `dm.stop()` in finally block
- `process.exit(1)` for validation errors
