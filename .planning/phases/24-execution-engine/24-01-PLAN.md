<task type="auto">
  <name>Task 2: Core execution engine</name>
  <files>packages/modules/collections/execution-engine.ts</files>
  <action>
Create packages/modules/collections/execution-engine.ts exporting a single function:

```ts
runCollection(opts: RunCollectionOpts): ExecutionRun
```

Where RunCollectionOpts contains:
- collection: Collection
- devices: Array<{ id: string; name: string; platform: string }>
- getAdapter: (platform: string) => PlatformAdapter | undefined
- onStepProgress: (run: ExecutionRun) => void  — called after each step completes on all devices
- onComplete: (run: ExecutionRun) => void
- onError: (run: ExecutionRun, error: Error) => void
- timeoutMs?: number (default 30000)

Implementation details:
1. Generate runId via crypto.randomUUID()
2. Build initial ExecutionRun with all steps in "pending" state, status "running"
3. Return the ExecutionRun object immediately (the execution runs asynchronously via a detached async IIFE — do NOT await it)
4. Iterate steps sequentially (for...of loop inside the async IIFE):
   a. For current step, resolve ActionDescriptor from action registry using step.actionId. If not found, mark all devices as "failed" with error "Unknown action: {actionId}", call onStepProgress, continue to next step.
   b. Set currentStepIndex on the run
   c. For each device in parallel (Promise.all), execute:
      - Get adapter via getAdapter(device.platform)
      - If adapter is undefined OR action.isSupported(adapter) returns false → mark device as "skipped", continue
      - Mark device as "running" with startedAt timestamp
      - Execute action.execute(adapter, device.id, step.params) wrapped in Promise.race with a timeout:
        - Use `Promise.race([action.execute(...), new Promise((_, reject) => setTimeout(() => reject(new Error("Timeout: step exceeded 30s")), timeoutMs))])`
      - On success: mark "success" with completedAt
      - On error (including timeout): mark "failed" with error.message and completedAt
   d. After all devices complete for this step, call onStepProgress(run)
5. After all steps: set run.status = "completed", run.completedAt = new Date().toISOString(), call onComplete(run)
6. Wrap the entire async IIFE in try/catch — on unexpected error: set run.status = "failed", call onError(run, error)

Import getActionDescriptors from ./action-registry.js to resolve actionIds. Build a Map<string, ActionDescriptor> at the start for O(1) lookup.

Do NOT import anything from @simvyn/server or fastify — this module must be usable from both server context and headless CLI.
  </action>
