**1. Add POST /execute to routes.ts:**

Add to collectionsRoutes function, after the existing DELETE route:

```
POST /:id/execute
Body: { deviceIds: string[] }
```

Handler logic:
- Read collections from storage, find by id (404 if not found)
- Validate deviceIds is a non-empty array (400 if invalid)
- Access `fastify.deviceManager` to resolve devices:
  - For each deviceId in body.deviceIds, find the device in `fastify.deviceManager.devices` via `d.id === deviceId || d.id.startsWith(deviceId)` (prefix matching)
  - If any device not found, return 404 with `{ error: "Device not found: {id}" }`
  - If any device is not booted (state !== "booted"), return 400 with `{ error: "Device {name} is not booted" }`
- Get adapter per device: `fastify.deviceManager.getAdapter(device.platform)`
- Call `runCollection()` from execution-engine.ts with:
  - collection
  - devices: resolved Device[] mapped to `{ id, name, platform }`
  - getAdapter: (platform) => fastify.deviceManager.getAdapter(platform)
  - onStepProgress: (run) => fastify.wsBroker.broadcast("collections", "step-progress", run)
  - onComplete: (run) => { activeRuns.delete(run.runId); fastify.wsBroker.broadcast("collections", "run-completed", run); }
  - onError: (run, err) => { activeRuns.delete(run.runId); fastify.wsBroker.broadcast("collections", "run-failed", { ...run, error: err.message }); }
- Store the returned ExecutionRun in a module-level `activeRuns` Map<string, ExecutionRun>
- Return `{ runId: run.runId }` with status 202

Also add a GET `/runs/:runId` endpoint that returns the ExecutionRun from the activeRuns Map (404 if not found). This lets clients poll status if WS is unavailable.

Declare `const activeRuns = new Map<string, ExecutionRun>()` at the top of collectionsRoutes.

**2. Create ws-handler.ts:**

Create packages/modules/collections/ws-handler.ts following the location/ws-handler.ts pattern:

```ts
export function registerCollectionsWsHandler(fastify: FastifyInstance) {
  const { wsBroker } = fastify;

  wsBroker.registerChannel("collections", (type, payload, socket, requestId) => {
    // Handle "cancel" type — future use, for now just acknowledge
    if (type === "cancel") {
      const { runId } = payload as any;
      // Cancel is a future feature — for now, acknowledge receipt
      wsBroker.send(socket, "collections", "cancel-ack", { runId, message: "Cancel not yet implemented" }, requestId);
      return;
    }
  });
}
```

Import `type {} from "@simvyn/server"` for Fastify decorator type augmentations (same pattern as routes.ts and location/ws-handler.ts).
