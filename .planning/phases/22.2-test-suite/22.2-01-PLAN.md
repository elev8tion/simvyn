<task type="auto">
  <name>Task 2: iOS adapter comprehensive tests</name>
  <files>packages/core/src/__tests__/ios-adapter.test.ts</files>
  <action>
  Create `packages/core/src/__tests__/ios-adapter.test.ts` using `node:test` and `node:assert`.

  **Mocking strategy:** Use `node:test`'s `mock` module to mock `verboseExec` and `verboseSpawn` from `../verbose-exec.js`. Since the iOS adapter imports these at module level, use `mock.module()` to replace the entire `../verbose-exec.js` module before importing the adapter. Create a helper that captures all calls to verboseExec/verboseSpawn so tests can assert on the exact command + args arrays.

  **Test structure:** One `describe` block per adapter method category:

  1. **isAvailable** — calls `xcrun simctl list devices --json`, returns true on success, false on error
  2. **listDevices** — returns parsed Device[] from simctl JSON output; test runtime key parsing (`iOS-17-5` → `iOS 17.5`), state mapping (Booted→booted, Shutdown→shutdown), filtering unavailable devices
  3. **boot** — calls `xcrun simctl boot <id>`, swallows "already booted" error
  4. **shutdown** — calls `xcrun simctl shutdown <id>`, swallows "current state: Shutdown" error
  5. **erase** — calls `xcrun simctl erase <id>`
  6. **setLocation** — calls `xcrun simctl location <id> set <lat>,<lon>`
  7. **clearLocation** — calls `xcrun simctl location <id> clear`
  8. **listApps** — mock plist→JSON conversion, verify `xcrun simctl listapps <id>` called
  9. **installApp** — verify `xcrun simctl install <id> <path>` for .app; for .ipa verify unzip + install + cleanup
  10. **uninstallApp** — `xcrun simctl uninstall <id> <bundleId>`
  11. **launchApp** — `xcrun simctl launch <id> <bundleId>`
  12. **terminateApp** — `xcrun simctl terminate <id> <bundleId>`
  13. **getAppInfo** — mock plist→JSON, verify `xcrun simctl appinfo <id> <bundleId>`, returns null on error
  14. **openUrl** — `xcrun simctl openurl <id> <url>`
  15. **screenshot** — `xcrun simctl io <id> screenshot <path>`
  16. **startRecording** — spawns `xcrun simctl io <id> recordVideo <path>`
  17. **stopRecording** — sends SIGINT to child process
  18. **getClipboard** — `xcrun simctl pbpaste <id>`
  19. **setClipboard** — spawns `xcrun simctl pbcopy <id>`, writes to stdin
  20. **addMedia** — `xcrun simctl addmedia <id> <path>`
  21. **setAppearance** — `xcrun simctl ui <id> appearance <mode>`
  22. **setStatusBar** — `xcrun simctl status_bar <id> override --time <val> ...` (verify flag mapping)
  23. **clearStatusBar** — `xcrun simctl status_bar <id> clear`
  24. **grantPermission** — `xcrun simctl privacy <id> grant <perm> <bundleId>`
  25. **revokePermission** — `xcrun simctl privacy <id> revoke <perm> <bundleId>`
  26. **resetPermissions** — `xcrun simctl privacy <id> reset all <bundleId>`
  27. **setLocale** — two calls: AppleLocale + AppleLanguages via `xcrun simctl spawn <id> defaults write`
  28. **setContentSize** — `xcrun simctl ui <id> content_size <size>`
  29. **setIncreaseContrast** — `xcrun simctl ui <id> increase_contrast enabled/disabled`
  30. **listDeviceTypes** — parses `xcrun simctl list devicetypes --json`
  31. **listRuntimes** — parses `xcrun simctl list runtimes --json`
  32. **createDevice** — `xcrun simctl create <name> <type> [runtime]`, returns trimmed stdout
  33. **cloneDevice** — `xcrun simctl clone <id> <name>`, returns trimmed stdout
  34. **renameDevice** — `xcrun simctl rename <id> <name>`
  35. **deleteDevice** — `xcrun simctl delete <id>`, wraps "Invalid device state" into user-friendly error
  36. **addKeychainCert** — writes cert to temp file, calls `xcrun simctl keychain <id> add-root-cert/add-cert <path>`, cleans up
  37. **resetKeychain** — `xcrun simctl keychain <id> reset`
  38. **collectBugReport** — `xcrun simctl diagnose -b --output <path>` with 300s timeout
  39. **capabilities** — returns expected array of PlatformCapability strings

  For each method, assert:
  - The correct binary (`xcrun`) is called
  - The exact argument array matches expected
  - Return values are correctly parsed/transformed
  - Error cases are handled (swallowed or re-thrown as appropriate)

  Use mock.module() at the top of the file to replace `../verbose-exec.js`. Track calls in an array like `const calls: { fn: string; args: any[] }[]`. Reset between tests via beforeEach.

  Note: For methods that use `plistToJson` (listApps, getAppInfo), mock verboseSpawn to return a process-like object that emits the JSON on stdout. The plist conversion is internal — test that the end-to-end flow from simctl output → parsed AppInfo works.
  </action>
