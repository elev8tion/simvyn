**Step A: Update module-store.ts — merge settings + dev-utils into device-settings.**

In `fetchModules()`, after the `fetch("/api/modules")` call and before storing the modules array, add a transformation:

```ts
// Merge "settings" and "dev-utils" into virtual "device-settings" module
const settingsIdx = data.findIndex((m: any) => m.name === "settings");
const devUtilsIdx = data.findIndex((m: any) => m.name === "dev-utils");
if (settingsIdx !== -1 && devUtilsIdx !== -1) {
  const settings = data[settingsIdx];
  const devUtils = data[devUtilsIdx];
  const merged = {
    name: "device-settings",
    description: settings.description,
    capabilities: [...(settings.capabilities || []), ...(devUtils.capabilities || [])],
  };
  // Replace settings entry with merged, remove dev-utils entry
  data[settingsIdx] = merged;
  data.splice(devUtilsIdx > settingsIdx ? devUtilsIdx : devUtilsIdx, 1);
}
```

This ensures the sidebar, command palette, and ModuleShell all see a single "device-settings" module. The URL will be `/device-settings`.
