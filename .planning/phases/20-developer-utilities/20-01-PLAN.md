**In packages/core/src/adapters/android.ts:**

Implement all 17 methods (everything except collectBugReport iOS path) inside `createAndroidAdapter()`, following the existing `execFileAsync("adb", ["-s", deviceId, ...])` pattern. Guard all methods with the `avd:` prefix check.

Port forwarding — NOTE: `adb forward` does NOT use `-s` between `adb` and `forward`. The syntax is `adb -s <serial> forward <local> <remote>`:
- `addForward`: `execFileAsync("adb", ["-s", deviceId, "forward", local, remote])`
- `removeForward`: `execFileAsync("adb", ["-s", deviceId, "forward", "--remove", local])`
- `listForwards`: `execFileAsync("adb", ["-s", deviceId, "forward", "--list"])` → parse output. Each line: `<serial> <local> <remote>`. Split by whitespace, return `{ local, remote }` for each line matching this device.
- `addReverse`: `execFileAsync("adb", ["-s", deviceId, "reverse", remote, local])`
- `removeReverse`: `execFileAsync("adb", ["-s", deviceId, "reverse", "--remove", remote])`
- `listReverses`: `execFileAsync("adb", ["-s", deviceId, "reverse", "--list"])` → same parsing as listForwards.

Display overrides — all use `adb shell wm`:
- `setDisplaySize`: `execFileAsync("adb", ["-s", deviceId, "shell", "wm", "size", `${width}x${height}`])`
- `resetDisplaySize`: `execFileAsync("adb", ["-s", deviceId, "shell", "wm", "size", "reset"])`
- `setDisplayDensity`: `execFileAsync("adb", ["-s", deviceId, "shell", "wm", "density", String(dpi)])`
- `resetDisplayDensity`: `execFileAsync("adb", ["-s", deviceId, "shell", "wm", "density", "reset"])`

Battery simulation — uses `adb shell dumpsys battery`:
- `setBattery`: For each provided option, run the appropriate command:
  - `level !== undefined` → `execFileAsync("adb", ["-s", deviceId, "shell", "dumpsys", "battery", "set", "level", String(level)])`
  - `status !== undefined` → `...["set", "status", String(status)]` (1=unknown, 2=charging, 3=discharging, 4=not_charging, 5=full)
  - `ac !== undefined` → `...["set", "ac", ac ? "1" : "0"]`
  - `usb !== undefined` → `...["set", "usb", usb ? "1" : "0"]`
- `unplugBattery`: `execFileAsync("adb", ["-s", deviceId, "shell", "dumpsys", "battery", "unplug"])`
- `resetBattery`: `execFileAsync("adb", ["-s", deviceId, "shell", "dumpsys", "battery", "reset"])`

Input injection — all use `adb shell input`:
- `inputTap`: `execFileAsync("adb", ["-s", deviceId, "shell", "input", "tap", String(x), String(y)])`
- `inputSwipe`: `execFileAsync("adb", ["-s", deviceId, "shell", "input", "swipe", String(x1), String(y1), String(x2), String(y2), ...(durationMs ? [String(durationMs)] : [])])`
- `inputText`: `execFileAsync("adb", ["-s", deviceId, "shell", "input", "text", text])` — NOTE: spaces in text must be escaped or the text split. adb input text doesn't handle spaces well. Replace spaces with `%s` or use `input text` per-word.
- `inputKeyEvent`: `execFileAsync("adb", ["-s", deviceId, "shell", "input", "keyevent", String(keyCode)])`

Bug reports:
- `collectBugReport`: Create output dir if needed (`~/.simvyn/bug-reports/`). Generate filename: `bugreport-${deviceId}-${Date.now()}.zip`. Run `execFileAsync("adb", ["-s", deviceId, "bugreport", outputPath])` — this may take 1-5 minutes, so use a longer timeout (5 min). Get file size with `fs.promises.stat`. Return `{ path, filename, size }`.

Add all 5 new capabilities to the `capabilities()` array: `"portForward"`, `"displayOverride"`, `"batterySimulation"`, `"inputInjection"`, `"bugReport"`.
