<task type="auto">
  <name>Task 1: DevUtilsPanel with 5 capability-gated sections</name>
  <files>packages/dashboard/src/panels/DevUtilsPanel.tsx</files>
  <action>
Create `DevUtilsPanel.tsx` following the SettingsPanel pattern exactly: fetch capabilities on device change, conditionally render sections based on capability booleans. Use `useDeviceStore` for selected device. Call `registerPanel("dev-utils", DevUtilsPanel)` at the bottom.

The capabilities fetch: `GET /api/modules/dev-utils/capabilities?deviceId={id}` returns `{ portForward, displayOverride, batterySimulation, inputInjection, bugReport }`.

If no device selected, show `glass-empty-state` with "Select a booted device to use developer utilities".

**Section 1: Port Forwarding** (shown when `capabilities.portForward`)

Glass card with title "Port Forwarding". Two sub-tabs or sub-sections: "Forward" (local→device) and "Reverse" (device→local).

For each direction:
- A form row: two text inputs (local port, remote port — e.g., `tcp:8080`, `tcp:3000`) + "Add" button.
- A table/list of active forwards fetched from `GET /api/modules/dev-utils/forward/list?deviceId={id}` (or `/reverse/list`). Each row shows local, remote, and a "Remove" button.
- On add: `POST /api/modules/dev-utils/forward/add` body `{ deviceId, local, remote }`. On remove: `POST /api/modules/dev-utils/forward/remove` body `{ deviceId, local }`.
- After add/remove, re-fetch the list.
- Use `toast.success/error` from sonner for feedback.

**Section 2: Display Overrides** (shown when `capabilities.displayOverride`)

Glass card with title "Display Overrides". Two groups:

Resolution:
- Two number inputs (width, height) + "Set Size" button. Calls `POST /api/modules/dev-utils/display/size` body `{ deviceId, width, height }`.
- "Reset Size" button. Calls `POST /api/modules/dev-utils/display/size/reset` body `{ deviceId }`.

Density:
- One number input (DPI) + "Set Density" button. Calls `POST /api/modules/dev-utils/display/density` body `{ deviceId, dpi }`.
- "Reset Density" button. Calls `POST /api/modules/dev-utils/display/density/reset` body `{ deviceId }`.

**Section 3: Battery Simulation** (shown when `capabilities.batterySimulation`)

Glass card with title "Battery Simulation".
- Range slider (0-100) for battery level + number input showing value.
- Select/dropdown for charging status: Unknown (1), Charging (2), Discharging (3), Not Charging (4), Full (5).
- Two checkboxes: "AC Power" and "USB Power".
- "Apply" button → `POST /api/modules/dev-utils/battery/set` body `{ deviceId, level, status, ac, usb }`.
- "Unplug" button → `POST /api/modules/dev-utils/battery/unplug` body `{ deviceId }`.
- "Reset" button → `POST /api/modules/dev-utils/battery/reset` body `{ deviceId }`.

**Section 4: Input Injection** (shown when `capabilities.inputInjection`)

Glass card with title "Input Injection". Four sub-sections using glass-tab-bar for switching between modes:

Tap mode: Two number inputs (x, y) + "Tap" button → `POST /api/modules/dev-utils/input/tap` body `{ deviceId, x, y }`.

Swipe mode: Four number inputs (x1, y1, x2, y2) + optional duration (ms) + "Swipe" button → `POST /api/modules/dev-utils/input/swipe` body `{ deviceId, x1, y1, x2, y2, durationMs }`.

Text mode: Text input + "Send Text" button → `POST /api/modules/dev-utils/input/text` body `{ deviceId, text }`.

Key Event mode: Text/number input for keyCode + "Send Key" button → `POST /api/modules/dev-utils/input/keyevent` body `{ deviceId, keyCode }`. Include a few common key presets as quick buttons: Home (3), Back (4), Menu (82), Power (26), Volume Up (24), Volume Down (25).

**Section 5: Bug Reports** (shown when `capabilities.bugReport`)

Glass card with title "Bug Reports".
- "Collect Bug Report" button → `POST /api/modules/dev-utils/bugreport/collect` body `{ deviceId }`.
- Show loading state with spinner during collection (this can take 1-5 minutes — show "Collecting... this may take several minutes").
- When complete, the response has `{ filename, downloadUrl, size }`. Show the result with filename, size, and a "Download" link pointing to downloadUrl.
- Keep a local state array of collected reports in the current session for easy access.

**Styling notes:**
- Use existing glass CSS classes: `glass-panel`, `glass-button`, `glass-button-primary`, `glass-button-destructive`, `glass-input`, `glass-tab-bar`, `glass-tab`, `glass-empty-state`, `glass-badge`.
- Section cards: `rounded-xl bg-bg-surface/10 border-b border-border p-4 space-y-3` (same as SettingsPanel sections).
- Section titles: `text-sm font-medium text-text-secondary uppercase tracking-wide`.
- Import Lucide icons as needed: `Network`, `Monitor`, `Battery`, `MousePointer`, `FileDown` (or similar — pick appropriate icons).
- No emojis anywhere.
