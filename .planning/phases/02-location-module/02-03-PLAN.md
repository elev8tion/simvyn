Create `packages/modules/location/parse-route.ts`:
- Import `DOMParser` from `@xmldom/xmldom`
- Import `gpx`, `kml` from `@tmcw/togeojson`
- Export `function parseRouteFile(content: string, format: "gpx" | "kml"): [number, number][]`
  - Create a DOM document: `new DOMParser().parseFromString(content, "text/xml")`
  - Convert to GeoJSON: `format === "gpx" ? gpx(doc) : kml(doc)`
  - Extract coordinates from all LineString and MultiLineString features in the FeatureCollection
  - GeoJSON coordinates are `[lon, lat]` — flip to `[lat, lon]` for our internal format
  - Also handle Point geometries (treat as single-waypoint route)
  - Return the `[lat, lon][]` array
  - If no coordinates found, throw an Error("No route coordinates found in file")

- Export `function detectFormat(filename: string): "gpx" | "kml"`
  - Check file extension: `.gpx` → "gpx", `.kml` → "kml"
  - Throw if unrecognized extension

Both functions are pure and stateless — no side effects.

```typescript
    .command("route <device> <file>")
    .description("Simulate GPS route from GPX/KML file")
    .option("-s, --speed <ms>", "Speed in m/s", "10")
    .action(async (deviceId: string, filePath: string, opts: { speed: string }) => {
      // ... implementation
    });

  location
    .command("clear <device>")
    .description("Clear GPS override on a device")
    .action(async (deviceId: string) => {
      // ... implementation
    });
}
```

**`set` command implementation:**
1. Parse lat/lng to numbers: `const lat = Number(latStr); const lng = Number(lngStr)`. Validate they're finite numbers — exit with error if not.
2. Dynamic import `@simvyn/core`: `const { createAvailableAdapters, createDeviceManager } = await import("@simvyn/core")`
3. Create adapters and device manager, refresh device list
4. Find target device by exact match or prefix match: `devices.find(d => d.id === deviceId || d.id.startsWith(deviceId))`
5. If not found, print error and `process.exit(1)`
6. Get adapter, check `adapter.setLocation` exists
7. Call `adapter.setLocation(target.id, lat, lng)`
8. Print success: `console.log(\`Set location on ${target.name}: ${lat}, ${lng}\`)`
9. Call `dm.stop()` to clean up

**`route` command implementation:**
1. Dynamic import `@simvyn/core` and `node:fs/promises`
2. Create adapters and device manager, refresh, find target device (same as set)
3. Read the route file: `await readFile(filePath, "utf-8")`
4. Import `detectFormat`, `parseRouteFile` from `./parse-route.js`
5. Parse: `const waypoints = parseRouteFile(content, detectFormat(filePath))`
6. Print: `console.log(\`Loaded ${waypoints.length} waypoints from ${filePath}\`)`
7. Parse speed: `const speedMs = Number(opts.speed)` (default 10 m/s)
8. Import `cumulativeDistances`, `interpolateAlongRoute` from `./geo.js`
9. Compute cumulative distances and total distance
10. Implement a tick loop (200ms interval) that:
    - Computes current position based on elapsed time * speedMs
    - Calls `adapter.setLocation!(target.id, lat, lon)` (for Android, this is the tick approach; for iOS this also works since `simctl location set` can be called repeatedly)
    - Prints progress: `\rProgress: ${Math.round(progress * 100)}% — ${lat.toFixed(6)}, ${lon.toFixed(6)}` using `process.stdout.write` for in-place update
    - When progress >= 1, clear interval, print "Route complete", call dm.stop()
11. Handle SIGINT to stop playback gracefully

NOTE: For the CLI route command, do NOT use the PlaybackEngine (which is designed for server-side with processManager). Instead, implement a simpler tick-based loop directly, since the CLI creates its own adapters and doesn't have a processManager. For iOS, calling `simctl location set` repeatedly works fine for CLI playback (the stdin pipe approach is an optimization only needed for the server's PlaybackEngine).

**`clear` command implementation:**
1. Same adapter/device setup as `set`
2. Call `adapter.clearLocation!(target.id)`
3. Print success message
4. Call `dm.stop()`
