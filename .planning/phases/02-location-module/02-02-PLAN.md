  <action>
Create `packages/modules/location/playback.ts` — the route simulation engine, migrated from sim-location's `server/playback.ts`.

Export `createPlaybackEngine(opts)` factory function that returns a `PlaybackEngine` instance.

Options type:
```typescript
interface PlaybackOptions {
  deviceId: string;
  platform: "ios" | "android";
  waypoints: [number, number][];  // [lat, lon] pairs
  speedMs: number;                // speed in m/s (default 10 = ~36 km/h)
  processManager: ProcessManager; // from fastify.processManager
  adapter: PlatformAdapter;       // for Android tick-based setLocation
  onPosition: (data: PlaybackPosition) => void;  // callback on each tick
  onComplete: () => void;
  onError: (error: Error) => void;
}

interface PlaybackPosition {
  lat: number;
  lon: number;
  progress: number;     // 0-1
  waypointIndex: number;
  speedMs: number;
}
```

Import `cumulativeDistances`, `interpolateAlongRoute` from `./geo.js`.
Import `ProcessManager` type from `@simvyn/core`.
Import `PlatformAdapter` from `@simvyn/types`.

PlaybackEngine interface:
```typescript
interface PlaybackEngine {
  start(): void;
  pause(): void;
  resume(): void;
  stop(): void;
  setSpeed(speedMs: number): void;
  state: "idle" | "playing" | "paused";
}
```

**iOS implementation** (when platform === "ios"):
- On `start()`: spawn `xcrun simctl location <deviceId> start --speed=<speedMs> --interval=1 -` using `processManager.spawn()`. The `-` flag means read waypoints from stdin.
- Write all waypoints to stdin as `lat,lon\n` pairs, one per line. Then call `child.stdin.end()`.
- Track the spawned child process. On `stop()` or `pause()`, kill the child process (`child.kill("SIGTERM")`).
- For position tracking (to update the UI), run a parallel timer that uses `interpolateAlongRoute` to compute the estimated current position based on elapsed time and speed. Tick every 200ms. Call `onPosition()` on each tick.
- On `resume()`, restart the simctl process from the current waypoint index forward (slice remaining waypoints).
- When the estimated distance traveled >= total route distance, call `onComplete()` and clean up.

**Android implementation** (when platform === "android"):
- On `start()`: use `setInterval` with 200ms tick. On each tick, compute current position using `interpolateAlongRoute` based on elapsed time * speedMs. Call `adapter.setLocation!(deviceId, lat, lon)`.
- Call `onPosition()` with the computed position on each tick.
- `pause()`: clear the interval, record elapsed distance.
- `resume()`: restart the interval from the paused distance.
- `stop()`: clear interval, call `adapter.clearLocation!(deviceId)`.
- When distance >= total route distance, call `onComplete()`.

`setSpeed(newSpeed)`: update the speed. For iOS, this requires restarting the simctl process. For Android, just update the speed variable used in tick calculations.

Track state as `"idle"` | `"playing"` | `"paused"`. Prevent invalid transitions (e.g., pause when idle).

CRITICAL: All timer references (setInterval/setTimeout) must be cleaned up on stop(). All child processes must go through processManager.spawn() so they're tracked for cleanup.

```typescript
import type { FastifyInstance } from "fastify";

```

Export `function registerLocationWsHandler(fastify: FastifyInstance)`.

Register channel "location" on `fastify.wsBroker`:

Handle these message types:

1. `"set-location"` — payload: `{ lat: number, lon: number, deviceIds?: string[] }`. If no deviceIds, set on all booted devices. For each device, get adapter and call `adapter.setLocation!(deviceId, lat, lon)`. Broadcast `"location-set"` with `{ lat, lon, results: Array<{ deviceId: string, success: boolean, error?: string }> }` to all location subscribers.

2. `"start-playback"` — payload: `{ deviceId: string, waypoints: [number, number][], speedMs?: number }`. Create a PlaybackEngine via `createPlaybackEngine()` with the device's platform and adapter. Store engine by deviceId in a `Map<string, PlaybackEngine>`. Wire `onPosition` to broadcast `"playback-position"` (payload: `{ deviceId, ...PlaybackPosition }`). Wire `onComplete` to broadcast `"playback-complete"` and remove from map. Call `engine.start()`. Send `"playback-started"` response to the requesting socket.

3. `"pause-playback"` — payload: `{ deviceId: string }`. Find engine in map, call `engine.pause()`. Broadcast `"playback-paused"`.

4. `"resume-playback"` — payload: `{ deviceId: string }`. Find engine, call `engine.resume()`. Broadcast `"playback-resumed"`.

5. `"stop-playback"` — payload: `{ deviceId: string }`. Find engine, call `engine.stop()`, remove from map. Broadcast `"playback-stopped"`.

6. `"set-speed"` — payload: `{ deviceId: string, speedMs: number }`. Find engine, call `engine.setSpeed(speedMs)`. Broadcast `"speed-changed"`.

For all messages, if device or engine not found, send error response back to the requesting socket via `wsBroker.send()`.
