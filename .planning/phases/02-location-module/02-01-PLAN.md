In `packages/types/src/device.ts`, add two optional methods to the `PlatformAdapter` interface:
- `setLocation?(deviceId: string, lat: number, lon: number): Promise<void>`
- `clearLocation?(deviceId: string): Promise<void>`

These are optional (with `?`) because not all future adapters may support them — but `setLocation` is already listed as a `PlatformCapability`.

In `packages/core/src/adapters/ios.ts`, add `setLocation` and `clearLocation` to the returned adapter object from `createIosAdapter()`:
- `setLocation(deviceId, lat, lon)`: call `execFileAsync("xcrun", ["simctl", "location", deviceId, "set", `${lat},${lon}`])`. Use the existing `execFileAsync` already imported.
- `clearLocation(deviceId)`: call `execFileAsync("xcrun", ["simctl", "location", deviceId, "clear"])`.

In `packages/core/src/adapters/android.ts`, add `setLocation` and `clearLocation` to the returned adapter object from `createAndroidAdapter()`:
- `setLocation(deviceId, lat, lon)`: call `execFileAsync("adb", ["-s", deviceId, "emu", "geo", "fix", String(lon), String(lat)])`. CRITICAL: Android's `geo fix` takes LONGITUDE FIRST, then latitude. This is intentional and correct.
- `clearLocation(deviceId)`: call `execFileAsync("adb", ["-s", deviceId, "emu", "geo", "fix", "0", "0"])` (reset to 0,0 — there is no dedicated "clear" for Android).

Do NOT change the `capabilities()` return arrays — `setLocation` is already listed in both.

Create `packages/modules/location/geo.ts` with these functions (migrated from sim-location's `utils/geo.ts`):
- `haversine(lat1: number, lon1: number, lat2: number, lon2: number): number` — returns distance in meters using the haversine formula. Use Earth radius = 6371000 meters.
- `interpolatePoint(p1: [number, number], p2: [number, number], fraction: number): [number, number]` — linear interpolation between two [lat, lon] points by fraction (0-1).
- `cumulativeDistances(route: [number, number][]): number[]` — returns array of cumulative haversine distances from start for each point in the route. First element is 0.
- `interpolateAlongRoute(route: [number, number][], distances: number[], targetDistance: number): [number, number]` — given cumulative distances, find the interpolated point at `targetDistance` meters along the route. Used by the playback engine for constant-speed route traversal.

All coordinate pairs are `[lat, lon]` format in the geo module. Export all functions.

Create `packages/modules/location/storage.ts`:
- Import `createModuleStorage` from `@simvyn/core`
- Define and export `SavedLocation` type: `{ id: string; name: string; lat: number; lon: number; createdAt: number }`
- Define and export `SavedRoute` type: `{ id: string; name: string; waypoints: [number, number][]; createdAt: number }`
- Export `locationStorage = createModuleStorage("location")`
- Export async helper functions: `getLocations(): Promise<SavedLocation[]>`, `saveLocation(loc: SavedLocation): Promise<void>` (appends to array), `deleteLocation(id: string): Promise<void>`, `getRoutes(): Promise<SavedRoute[]>`, `saveRoute(route: SavedRoute): Promise<void>`, `deleteRoute(id: string): Promise<void>`. These read/write the `"locations"` and `"routes"` keys respectively.
