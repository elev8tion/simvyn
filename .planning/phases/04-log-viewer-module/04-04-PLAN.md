Import `LogEntry` and `LogLevel` from `@simvyn/types`.

`addBatch`: append entries, if total > 50_000 slice to keep last 50_000.
`clear`: reset entries to empty array.
`setStreaming`: set streamDeviceId and isStreaming flag.

Export a `useLogStore` hook and a `selectFilteredEntries` selector function:
```typescript
export function selectFilteredEntries(state: LogStore): LogEntry[] {
  const levels = ["verbose", "debug", "info", "warning", "error", "fatal"];
  const minIdx = levels.indexOf(state.minLevel);
  let filtered = state.entries;

  // Level filter
  if (minIdx > 0) {
    filtered = filtered.filter(e => levels.indexOf(e.level) >= minIdx);
  }

  // Process filter
  if (state.processFilter) {
    const pf = state.processFilter.toLowerCase();
    filtered = filtered.filter(e => e.processName.toLowerCase().includes(pf));
  }

  // Search pattern (try as regex, fall back to includes)
  if (state.searchPattern) {
    try {
      const re = new RegExp(state.searchPattern, "i");
      filtered = filtered.filter(e => re.test(e.message) || re.test(e.processName));
    } catch {
      const sp = state.searchPattern.toLowerCase();
      filtered = filtered.filter(e =>
        e.message.toLowerCase().includes(sp) || e.processName.toLowerCase().includes(sp)
      );
    }
  }

  return filtered;
}
```

**LogList.tsx** — Scrollable log entry display:

A `<div>` with `overflow-y: auto` and `max-height` set to fill available space. Receives `entries: LogEntry[]` as prop.

Each log entry rendered as a single row:
- Timestamp (dimmed text, `text-text-secondary`, show time portion only: `HH:MM:SS.mmm`)
- Level badge: colored pill/span with level name. Color map:
  - verbose: gray (`text-gray-500`)
  - debug: cyan (`text-cyan-400`)
  - info: white/default (`text-text-primary`)
  - warning: yellow (`text-yellow-400`)
  - error: red (`text-red-400`)
  - fatal: magenta/pink (`text-pink-400`) with bold
- Process name (dimmed, truncated)
- Message (main text)

Use monospace font (`font-mono text-xs`) for the entire log list — this is a log viewer, not prose.

Auto-scroll to bottom when new entries arrive UNLESS user has scrolled up. Implementation: track a `userScrolledUp` ref. On scroll event, check if `scrollTop + clientHeight < scrollHeight - threshold`. If user is at bottom, auto-scroll on new entries.

Render efficiently: use `key={index}` (entries are append-only). If performance becomes an issue, virtualization is a Phase 5 concern per research doc — don't over-engineer now.
