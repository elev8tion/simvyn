Create `packages/modules/log-viewer/log-streamer.ts` — the core LogStreamer class:

**LogStreamer interface:**
- Constructor takes: `{ deviceId, platform, processManager, onFlush, flushIntervalMs?, maxBufferSize? }`
  - `processManager` typed as `{ spawn(command: string, args: string[], opts?: any): ChildProcess }` (SpawnCapable-style interface, not importing core ProcessManager directly to avoid server/core type mismatch — same pattern as PlaybackEngine)
  - `onFlush: (entries: LogEntry[]) => void` — callback when batch is ready
  - `flushIntervalMs` defaults to 150
  - `maxBufferSize` defaults to 10_000
- `start()`: Spawns the platform-appropriate child process, sets up readline on stdout, starts flush timer
- `stop()`: Kills child process, clears flush timer, flushes remaining entries
- `isRunning` getter
- `history` getter — returns the capped history buffer (readonly)

**iOS spawn:** `xcrun simctl spawn <deviceId> log stream --style ndjson --level debug`
- Parse each line with `parseIosLogLine()`: `JSON.parse(line)`, skip if `eventType !== "logEvent"`, map fields to LogEntry
- `mapIosLevel()`: Default→info, Info→info, Debug→debug, Error→error, Fault→fatal, else→verbose
- Extract processName from `processImagePath` using basename (last `/`-delimited segment)

**Android spawn:** `adb -s <deviceId> logcat -v json`
- Guard: if `deviceId.startsWith("avd:")` throw Error("Device must be booted")
- Parse each line with `parseAndroidLogLine()`: `JSON.parse(line)`, map fields to LogEntry
- `mapAndroidPriority()`: 2→verbose, 3→debug, 4→info, 5→warning, 6→error, 7→fatal, else→info
- Use `obj.tag` as processName, `new Date().toISOString()` as timestamp fallback if no timestamp field

**Buffering:**
- Accumulate parsed LogEntry objects in a pending array
- `setInterval` flushes: if pending.length > 0, call `onFlush([...pending])`, clear pending
- Also push each entry to a history array (capped at maxBufferSize — shift oldest when full)
- On stop(), flush any remaining pending entries, then kill child

**Error handling:**
- readline `close` event: mark as not running
- child `error` event: mark as not running
- JSON.parse failures: silently skip the line (return null from parse functions)

Use `import { createInterface } from "node:readline"` for line-by-line stdout parsing. Use `import type { ChildProcess } from "node:child_process"`.

Export: `createLogStreamer(opts)` factory function (follows existing factory pattern) and the parse/map helper functions (exported for CLI reuse).
