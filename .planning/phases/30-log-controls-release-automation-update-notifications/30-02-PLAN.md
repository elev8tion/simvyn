<task type="auto">
  <name>Task 2: Add commit-based changelog to GitHub Release</name>
  <files>.github/workflows/release.yml</files>
  <action>
    Modify the `github-release` job in `.github/workflows/release.yml`:

    1. After the checkout step, add a step to generate the changelog body:
       ```yaml
       - name: Generate changelog
         id: changelog
         run: |
           # Find the previous tag
           PREV_TAG=$(git tag --sort=-version:refname | grep '^v' | sed -n '2p')
           CURRENT_TAG="${GITHUB_REF_NAME}"
           
           if [ -z "$PREV_TAG" ]; then
             # First release — list all commits
             CHANGES=$(git log --oneline --pretty=format:"- %s" "$CURRENT_TAG")
           else
             CHANGES=$(git log --oneline --pretty=format:"- %s" "${PREV_TAG}..${CURRENT_TAG}")
           fi
           
           # Write to file for multi-line output
           echo "$CHANGES" > /tmp/changelog.md
       ```

    2. Update the `softprops/action-gh-release@v2` step:
       - Remove `generate_release_notes: true`
       - Add `body_path: /tmp/changelog.md`
       - Keep the `with:` block

    The final github-release job should look like:
    ```yaml
    github-release:
      needs: publish
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
          with:
            fetch-depth: 0

        - name: Generate changelog
          run: |
            PREV_TAG=$(git tag --sort=-version:refname | grep '^v' | sed -n '2p')
            CURRENT_TAG="${GITHUB_REF_NAME}"
            if [ -z "$PREV_TAG" ]; then
              CHANGES=$(git log --oneline --pretty=format:"- %s" "$CURRENT_TAG")
            else
              CHANGES=$(git log --oneline --pretty=format:"- %s" "${PREV_TAG}..${CURRENT_TAG}")
            fi
            echo "$CHANGES" > /tmp/changelog.md

        - name: Create GitHub Release
          uses: softprops/action-gh-release@v2
          with:
            body_path: /tmp/changelog.md
    ```

    Note: Add `fetch-depth: 0` to the checkout step so git log can access the full history and previous tags.
  </action>
  <verify>
    YAML is valid. The github-release job has `fetch-depth: 0` on checkout, a changelog generation step using `git log`, and `body_path` instead of `generate_release_notes`.
  </verify>
  <done>
