  <files>

  <action>
    In LogPanel.tsx:
    1. Read `isPaused` from the log store: `const isPaused = useLogStore((s) => s.isPaused)`
    2. Add a new useEffect watching `isPaused` and `selectedDeviceId`:
       ```
       useEffect(() => {
         if (!selectedDeviceId) return;
         if (isPaused) {
           // Pause: stop the stream
           send({ channel: "logs", type: "stop-stream", payload: { deviceId: selectedDeviceId } });
         } else {
           // Resume: clear old entries, start fresh stream + load history
           // Only resume if we were previously paused (avoid double-start on mount)
           // Check if store was previously streaming for this device
           clear();
           send({ channel: "logs", type: "start-stream", payload: { deviceId: selectedDeviceId } });
           send({ channel: "logs", type: "get-history", payload: { deviceId: selectedDeviceId, limit: 500 } });
         }
       }, [isPaused]);
       ```
       IMPORTANT: The dependency array should ONLY contain `isPaused` — not `selectedDeviceId`, since device changes are already handled by the existing device change effect. This effect should only fire when the user explicitly toggles pause. Add a ref (`wasPaused`) initialized to false, and only send resume messages when `wasPaused.current === true && !isPaused`. Set `wasPaused.current = isPaused` at the end of the effect. This prevents double-start on mount.
    3. In the existing device change effect (the one with `prevDeviceRef`), add: when the device changes, reset isPaused via `useLogStore.getState().resume()` — this ensures switching devices auto-resumes. Call this BEFORE the start-stream send.
    4. In the `handleStreamStarted` callback, ensure it only calls `setStreaming` when not paused.
    5. In the `handleStreamStopped` callback, leave as-is (it already sets streaming to null).
  </action>
