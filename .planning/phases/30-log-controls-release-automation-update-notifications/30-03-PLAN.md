  <files>packages/server/src/app.ts</files>
  <action>
    In `packages/server/src/app.ts`, add a new route after the existing `/api/health` route:

    ```typescript
    fastify.get("/api/update-check", async () => {
      try {
        const controller = new AbortController();
        const timeout = setTimeout(() => controller.abort(), 3000);
        const res = await fetch("https://registry.npmjs.org/simvyn/latest", {
          signal: controller.signal,
        });
        clearTimeout(timeout);
        if (!res.ok) return { current: version, latest: null, needsUpdate: false };
        const data = (await res.json()) as { version: string };
        const latest = data.version;
        // Simple semver comparison
        const l = latest.split(".").map(Number);
        const c = version.split(".").map(Number);
        let needsUpdate = false;
        for (let i = 0; i < 3; i++) {
          if ((l[i] ?? 0) > (c[i] ?? 0)) { needsUpdate = true; break; }
          if ((l[i] ?? 0) < (c[i] ?? 0)) break;
        }
        return { current: version, latest, needsUpdate };
      } catch {
        return { current: version, latest: null, needsUpdate: false };
      }
    });
    ```

    The `version` variable: check how the server receives it. Look at `startServer()` function signature — it likely receives options including version. If `version` is not already available in the server context, check `fastify.version` or look at how the CLI passes version info. The CLI's `start.ts` calls `startServer()` with options — if version isn't passed, add it to the `StartOptions` interface and pass `ctx.version` from the CLI.

    Key points:
    - 3-second timeout to prevent slow responses
    - Returns `{ current, latest, needsUpdate }` — never throws
    - Uses the same npm registry API as the CLI's `update-check.ts`
    - No caching needed — this endpoint is called once per dashboard page load
