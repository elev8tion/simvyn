Replace the empty `cli(program)` method in manifest.ts with full subcommand implementations. Follow the location module's CLI pattern exactly: each action does `const { createAvailableAdapters, createDeviceManager } = await import("@simvyn/core")`, creates adapters, creates DeviceManager, finds device by ID or prefix match, gets adapter, checks method support, performs operation, calls `dm.stop()`.

Register parent command: `const app = program.command("app").description("App management commands");`

**Subcommand: `app list <device>`**
- Option: `--type <filter>` (default "all") — filter by "user", "system", or "all"
- Find device, check booted state, call `adapter.listApps(deviceId)`
- Filter by type if specified
- Output as formatted table: columns for Bundle ID, Name, Version, Type
- Use `console.log` with padded columns (follow device list CLI formatting pattern)
- If no apps found, print "No apps found"

**Subcommand: `app install <device> <path>`**
- Validate that the file exists at the given path (use `stat` from `node:fs/promises`)
- Find device, check booted state, call `adapter.installApp(deviceId, path)`
- Print success message: "Installed <filename> on <deviceName>"

**Subcommand: `app uninstall <device> <bundle-id>`**
- Find device, check booted state, call `adapter.uninstallApp(deviceId, bundleId)`
- Print: "Uninstalled <bundleId> from <deviceName>"

**Subcommand: `app launch <device> <bundle-id>`**
- Find device, check booted state, call `adapter.launchApp(deviceId, bundleId)`
- Print: "Launched <bundleId> on <deviceName>"

**Subcommand: `app terminate <device> <bundle-id>`**
- Find device, check booted state, call `adapter.terminateApp(deviceId, bundleId)`
- Print: "Terminated <bundleId> on <deviceName>"

**Subcommand: `app info <device> <bundle-id>`**
- Find device, check booted state, call `adapter.getAppInfo(deviceId, bundleId)`
- If null, print "App not found" and exit 1
- Print key-value pairs: Bundle ID, Name, Version, Type, Data Container, App Path

**Subcommand: `app clear-data <device> <bundle-id>`**
- Find device, check booted state, call `adapter.clearAppData(deviceId, bundleId)`
- If method not supported (iOS), print error: "Clear data not supported for <platform>" and exit 1
- Print: "Cleared data for <bundleId> on <deviceName>"

**Common patterns for all subcommands:**
- Check `device.state === "booted"` before any operation; print "Device must be booted" and exit 1 if not
- Check adapter method exists (e.g., `if (!adapter?.listApps)`) before calling; print "Not supported for <platform>" and exit 1
- Wrap operation in try/catch; print error message on failure and exit 1
- Always call `dm.stop()` before exiting (use try/finally)
