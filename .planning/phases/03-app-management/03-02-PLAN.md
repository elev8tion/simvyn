**routes.ts** — Register `@fastify/multipart` scoped to this plugin. Create these endpoints:

- `GET /list/:deviceId` — Resolve device from `fastify.deviceManager.devices`, get adapter, call `adapter.listApps(deviceId)`. Return array of AppInfo. Return 404 if device not found, 400 if adapter doesn't support listApps, 400 if device state !== "booted".

- `POST /install/:deviceId` — Use `req.file()` from multipart to receive uploaded file. Save to temp dir using `pipeline` + `createWriteStream`. Call `adapter.installApp(deviceId, tmpPath)`. Clean up temp in `finally`. Return `{ success: true }`. Limit file size to 500MB.

- `POST /uninstall` — Body: `{ deviceId, bundleId }`. Call `adapter.uninstallApp(deviceId, bundleId)`.

- `POST /launch` — Body: `{ deviceId, bundleId }`. Call `adapter.launchApp(deviceId, bundleId)`.

- `POST /terminate` — Body: `{ deviceId, bundleId }`. Call `adapter.terminateApp(deviceId, bundleId)`.

- `GET /info/:deviceId/:bundleId` — Call `adapter.getAppInfo(deviceId, bundleId)`. Return AppInfo or 404.

- `POST /clear-data` — Body: `{ deviceId, bundleId }`. Call `adapter.clearAppData(deviceId, bundleId)`. Return 400 if not supported.

Each route follows the location routes pattern: find device, get adapter, check method support, try/catch with error response.

**ws-handler.ts** — Register channel "apps" on the WsBroker:

Handle these message types:
- `list-apps`: `{ deviceId }` — call `adapter.listApps()`, reply with `app-list` containing `{ deviceId, apps: AppInfo[] }`.
- `install-app`: `{ deviceId, path }` — for server-side path installs (not file upload). Call `adapter.installApp()`, broadcast `app-installed` with `{ deviceId, path }`.
- `uninstall-app`: `{ deviceId, bundleId }` — call `adapter.uninstallApp()`, broadcast `app-uninstalled`.
- `launch-app`: `{ deviceId, bundleId }` — call `adapter.launchApp()`, broadcast `app-launched`.
- `terminate-app`: `{ deviceId, bundleId }` — call `adapter.terminateApp()`, broadcast `app-terminated`.
- `clear-data`: `{ deviceId, bundleId }` — call `adapter.clearAppData()`, broadcast `data-cleared`.

For errors, send `error` type to the requesting socket with `{ message }`.

Follow the location ws-handler pattern: import `type {} from "@simvyn/server"` for Fastify augmentations, destructure `wsBroker` and `deviceManager` from fastify.
