**app-store.ts** — Zustand store for app management state:

```typescript
interface AppInfo {
  bundleId: string;
  name: string;
  version: string;
  type: "user" | "system";
  dataContainer?: string;
  appPath?: string;
}

interface AppStore {
  apps: AppInfo[];
  loading: boolean;
  error: string | null;
  filter: "all" | "user" | "system";
  setApps(apps: AppInfo[]): void;
  setLoading(loading: boolean): void;
  setError(error: string | null): void;
  setFilter(filter: "all" | "user" | "system"): void;
  fetchApps(deviceId: string): Promise<void>;
}
```

`fetchApps` calls `fetch("/api/modules/apps/list/" + deviceId)`, parses JSON response, calls `setApps`. Sets `loading` true at start, false at end. Catches errors into `setError`.

**AppList.tsx** — Renders the app list as a table/list inside a glass-panel:

- Uses `useAppStore` for apps, loading, error, filter state
- Filter bar at top with three buttons: All / User / System (highlighted when active)
- Filter apps by `filter` state (if "all" show everything)
- Table with columns: Name, Bundle ID, Version, Type
- Each row has an actions column (rendered by AppActions)
- If loading, show a loading indicator
- If error, show error message
- If no apps after filtering, show "No apps found" message
- Use Tailwind classes following the glass-panel design system (dark bg, muted text, accent hover)

**AppActions.tsx** — Action buttons per app row:

- Props: `{ app: AppInfo, deviceId: string, onRefresh: () => void }`
- Launch button (play icon or "Launch" text) — POST to `/api/modules/apps/launch` with `{ deviceId, bundleId: app.bundleId }`
- Terminate button (stop icon or "Stop" text) — POST to `/api/modules/apps/terminate` with `{ deviceId, bundleId: app.bundleId }`
- Uninstall button (trash icon or "Uninstall" text) — POST to `/api/modules/apps/uninstall` with `{ deviceId, bundleId: app.bundleId }`, calls onRefresh after success
- Clear Data button (only shown if app.type === "user") — POST to `/api/modules/apps/clear-data` with `{ deviceId, bundleId: app.bundleId }`, calls onRefresh after success
- Each button: try/catch, show error via console.error (toast integration can come in Phase 5)
- Small icon-style buttons with hover states using Tailwind

**InstallDropZone.tsx** — Drag-and-drop file upload component:

- Accepts `.ipa` and `.apk` files
- Props: `{ deviceId: string, onInstallComplete: () => void }`
- Uses `onDragOver`, `onDragLeave`, `onDrop` event handlers on a div
- On drop: validate file extension is `.ipa` or `.apk`. Create `FormData`, append the file. POST to `/api/modules/apps/install/${deviceId}` with the FormData body (no Content-Type header — let browser set multipart boundary). Show uploading state while in progress. On success call `onInstallComplete`. On error show error message.
- Also include a file input button as alternative to drag-and-drop
- Visual: dashed border area with "Drop IPA or APK here to install" text. Highlight border on drag-over. Show progress/status text during upload.
- Tailwind styling: `border-2 border-dashed border-white/20 rounded-lg p-6 text-center`, drag-over state: `border-accent/60 bg-white/5`

**AppPanel.tsx** — Main panel component:

- Uses `useDeviceStore` to get devices and selected device
- Auto-selects first booted device if none selected (same pattern as LocationPanel)
- Device selector dropdown at top (if you want per-panel device selection) OR use the global device selector from TopBar. Prefer: use a local device dropdown so users can manage apps on a specific device independently of the global selector.
- When device is selected and booted, fetch apps via `useAppStore.fetchApps(deviceId)`
- Refetch when device changes
- Layout:
  - Top section: device dropdown + filter buttons
  - Middle section: InstallDropZone (collapsible or always visible, compact)
  - Main section: AppList (scrollable)
- Listen to WS channel "apps" for `app-installed`, `app-uninstalled`, `data-cleared` events to auto-refresh the app list
- Use `useWsListener("apps", "app-installed", handleRefresh)` pattern
