**iOS adapter (`packages/core/src/adapters/ios.ts`):**

Import `AppInfo` from `@simvyn/types`. Add a helper `plistToJson(plistStr: string): Promise<string>` that spawns `plutil -convert json -r -o - -- -` and pipes `plistStr` to stdin (use `spawn` from `node:child_process`, NOT `execFile` with `input` option — use the stdin pipe pattern from the research file).

Implement these methods on the adapter object returned by `createIosAdapter()`:

- `listApps(deviceId)`: Run `xcrun simctl listapps <deviceId>`, pipe stdout through `plistToJson()`, parse JSON. Map each entry to `AppInfo` using `CFBundleIdentifier` → bundleId, `CFBundleDisplayName ?? CFBundleName ?? CFBundleIdentifier` → name, `CFBundleVersion` → version, `ApplicationType.toLowerCase()` → type, `DataContainer` (strip `file://` prefix) → dataContainer, `Path` → appPath.

- `installApp(deviceId, appPath)`: If path ends with `.ipa`, unzip to a temp dir (`mkdtemp` in `os.tmpdir()`), find `Payload/*.app`, use that as install path. Run `xcrun simctl install <deviceId> <installPath>`. Clean up temp dir in `finally` block. If path ends with `.app` or is a directory, use directly.

- `uninstallApp(deviceId, bundleId)`: Run `xcrun simctl uninstall <deviceId> <bundleId>`.

- `launchApp(deviceId, bundleId)`: Run `xcrun simctl launch <deviceId> <bundleId>`.

- `terminateApp(deviceId, bundleId)`: Run `xcrun simctl terminate <deviceId> <bundleId>`.

- `getAppInfo(deviceId, bundleId)`: Run `xcrun simctl appinfo <deviceId> <bundleId>`, pipe through `plistToJson()`, parse and return single `AppInfo`. Return `null` on error (catch).

- `clearAppData`: Set to `undefined` (iOS has no simctl command for this).

**Android adapter (`packages/core/src/adapters/android.ts`):**

Import `AppInfo` from `@simvyn/types`.

- `listApps(deviceId)`: Run `adb -s <deviceId> shell pm list packages -f -3`. Parse output lines matching `package:(.+?)=([^\s]+)`. Return array of `AppInfo` with bundleId from capture group 2, name = bundleId, version = "unknown", type = "user", appPath from capture group 1.

- `installApp(deviceId, appPath)`: Run `adb -s <deviceId> install <appPath>`. No IPA handling needed — APKs install directly.

- `uninstallApp(deviceId, bundleId)`: Run `adb -s <deviceId> uninstall <bundleId>`.

- `launchApp(deviceId, bundleId)`: Run `adb -s <deviceId> shell monkey -p <bundleId> -c android.intent.category.LAUNCHER 1`. Use `monkey` (not `am start`) to avoid needing the launcher activity name.

- `terminateApp(deviceId, bundleId)`: Run `adb -s <deviceId> shell am force-stop <bundleId>`.

- `getAppInfo(deviceId, bundleId)`: Run `adb -s <deviceId> shell dumpsys package <bundleId>`. Parse with regex: `versionName=(.+)` → version, `versionCode=(\d+)` → append to version as ` (code)`, `dataDir=(.+)` → dataContainer, `codePath=(.+)` → appPath. Return null on error (catch).

- `clearAppData(deviceId, bundleId)`: Run `adb -s <deviceId> shell pm clear <bundleId>`.

**CRITICAL:** All commands must use `execFileAsync` (never `exec`). The `plistToJson` helper must use `spawn` with stdin pipe. Guard Android app commands against `avd:` prefix device IDs — throw an error like "Device must be booted for app operations" if deviceId starts with `avd:`.
