<task type="auto">
  <name>Task 1: Collection CRUD routes and action catalog endpoint</name>
  <files>packages/modules/collections/routes.ts</files>
  <action>
Create `packages/modules/collections/routes.ts` exporting `collectionsRoutes` as an async Fastify plugin function.

**Storage:** At the top of the plugin, initialize storage with `createModuleStorage("collections")` from `@simvyn/core`. Collections are stored under key `"collections"` as an array: `Collection[]`.

Helper: `readCollections()` reads from storage, returns `[]` if null. `writeCollections(collections)` writes the array.

**Routes (all prefixed by module-loader under `/api/modules/collections`):**

1. **GET `/actions`** — Returns the action catalog for the builder UI. Import `getActionDescriptors` from `./action-registry.js`. Map each ActionDescriptor to a serializable form (omit `execute` and `isSupported` functions). Return: `{ id, label, description, module, params }[]`. This endpoint does NOT need a device — it returns all possible actions regardless of platform.

2. **GET `/`** — List all collections. Returns the full collections array (sorted by updatedAt descending).

3. **GET `/:id`** — Get single collection by ID. Return 404 if not found.

4. **POST `/`** — Create collection. Body: `{ name: string, description?: string, steps?: CollectionStep[] }`. Generate UUID `id` with `crypto.randomUUID()`, set `schemaVersion: 1`, set `createdAt` and `updatedAt` to `new Date().toISOString()`, default `steps` to `[]`. Validate name is non-empty string (return 400 if invalid). Append to collections array and persist. Return 201 with the created collection.

5. **PUT `/:id`** — Update collection. Body: `{ name?: string, description?: string, steps?: CollectionStep[] }`. Find collection by ID, return 404 if not found. Merge provided fields (only update fields present in body), update `updatedAt`. Persist and return updated collection.

6. **POST `/:id/duplicate`** — Duplicate collection. Find by ID, return 404 if not found. Create a deep copy with new `crypto.randomUUID()` id, name set to `"{original.name} (Copy)"`, new `createdAt`/`updatedAt`, same `schemaVersion: 1`. Each step in the copy gets a new UUID id. Append and persist. Return 201 with the new collection.

7. **DELETE `/:id`** — Delete collection. Find by ID, return 404 if not found. Filter it out of the array, persist. Return `{ success: true }`.

Use `import type {} from "@simvyn/server"` for Fastify decorator type augmentations (same pattern as device-settings/routes.ts), though this module does NOT need deviceManager — it's a meta-module using only storage.

Follow the route typing pattern from device-settings: `fastify.post<{ Body: {...} }>("/path", handler)` and `fastify.get<{ Params: {...} }>("/:id", handler)`.
