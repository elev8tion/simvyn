<task type="auto">
  <name>Task 1: Upgrade device-store.ts for multi-select</name>
  <files>packages/dashboard/src/stores/device-store.ts</files>
  <action>
  Rewrite the DeviceStore interface and implementation:

  1. Replace `selectedDeviceId: string | null` with `selectedDeviceIds: string[]` (empty array = no selection)
  2. Remove `broadcastMode: boolean` and `toggleBroadcast()` — replaced by multi-select
  3. Keep `selectDevice(id: string)` — sets `selectedDeviceIds` to `[id]` (single-select behavior)
  4. Add `toggleDevice(id: string)` — if id is in array, remove it (guard: don't remove last); if not, add it (multi-select behavior)
  5. Add `truncateToFirst()` — sets `selectedDeviceIds` to `[selectedDeviceIds[0]]` if length > 1. Used when switching away from multi-select module.
  6. Update `selectedDevice()` getter — return device matching `selectedDeviceIds[0]`
  7. Update `setDevices()` — auto-select logic uses `selectedDeviceIds[0]` instead of `selectedDeviceId`. If current selection becomes invalid, set to `[devices[0]?.id]` (filter out undefined).

  Keep the same Zustand `create` pattern. Export type should remain `useDeviceStore`.
  </action>
  <verify>TypeScript compiles: `npx tsc --noEmit -p packages/dashboard/tsconfig.json` (check for device-store errors only — other panels will break until Plan 02)</verify>
  <done>device-store.ts exports `selectedDeviceIds`, `selectDevice`, `toggleDevice`, `truncateToFirst`. No `broadcastMode`.</done>

<task type="auto">
  <name>Task 2: Refactor DeviceSelector for adaptive single/multi mode</name>
  <files>packages/dashboard/src/components/DeviceSelector.tsx</files>
  <action>
  Refactor DeviceSelector to detect the active module and switch between single-select and multi-select UI:

  1. Import `useModuleStore` from `../stores/module-store`
  2. Define `MULTI_SELECT_MODULES = new Set(["location"])` at module scope
  3. Read `activeModule` from `useModuleStore(s => s.activeModule)`
  4. Derive `isMultiSelect = MULTI_SELECT_MODULES.has(activeModule ?? "")`
  5. Read `selectedDeviceIds`, `selectDevice`, `toggleDevice` from `useDeviceStore`

  **Single-select mode** (default):
  - Clicking a device calls `selectDevice(id)` — replaces entire selection
  - Selected device shows a checkmark or highlighted background
  - Label shows selected device name or "No devices"

  **Multi-select mode** (location):
  - Each device row has a checkbox-style indicator (filled circle = selected, empty = not)
  - Clicking a device calls `toggleDevice(id)` — adds/removes from selection
  - Label shows "N devices selected" when multiple, device name when single
  - Guard in store prevents deselecting last device

  **Auto-truncate on mode switch:**
  - Add a `useEffect` watching `isMultiSelect`. When `isMultiSelect` becomes false and `selectedDeviceIds.length > 1`, call `truncateToFirst()`.

  Remove all `broadcastMode` references. Remove the "All devices (broadcast)" toggle row.

  Keep the existing glass dropdown styling, platform grouping, state indicator dots.
  </action>
  <verify>`npm run build` succeeds (DeviceSelector compiles). Open dashboard — TopBar selector opens, shows devices, clicking selects one.</verify>
  <done>DeviceSelector shows single-select for all modules except location. When location is active, shows multi-select with checkboxes. Switching away truncates to first.</done>
</task>
