  8. **Keep** all existing guard logic (`if (!selectedDeviceId) return <empty-state>`) — it still works since the global store may have no selection

  The panels already have empty-state rendering for no device. Just make sure the guard stays.

  **Important for LogPanel specifically:** LogPanel has a `useEffect` that sends `start-stream` / `stop-stream` WS messages when `selectedDeviceId` changes. This effect should still work — it will now react to the global store changing instead of local state. Verify the effect's dependency array includes `selectedDeviceId`.
