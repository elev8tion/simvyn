Create the multi-step flow engine for the command palette.

**types.ts** — Define the step-based action system:
```ts
type StepType = "device-select" | "confirm" | "parameter" | "execute";

interface Step {
  id: string;
  type: StepType;
  label: string;  // breadcrumb label for this step
}

interface DeviceSelectStep extends Step {
  type: "device-select";
  multi: boolean;  // single vs multi device selection
  filter?: (device: Device) => boolean;  // e.g. iOS-only for erase
}

interface ConfirmStep extends Step {
  type: "confirm";
  message: string;  // "Are you sure you want to erase {deviceName}?"
  destructive?: boolean;
}

interface MultiStepAction {
  id: string;
  label: string;
  description: string;
  icon: React.ReactNode;
  steps: Step[];  // ordered steps to walk through
  execute: (context: StepContext) => Promise<void>;  // final execution with collected data
}

interface StepContext {
  selectedDeviceIds: string[];  // devices picked in DevicePicker
  params: Record<string, unknown>;  // any parameters collected
}
```

**DevicePicker.tsx** — Inline device picker rendered inside the palette:
- Reads `useDeviceStore.getState().devices` for the full device list (only booted devices)
- Renders each device as a `Command.Item` with device name, platform badge (iOS/Android), and OS version
- Supports single-select (click confirms) and multi-select (checkboxes, "Apply" button at bottom) modes based on `multi` prop
- Does NOT mutate global `selectedDeviceIds` — collects selection locally and calls `onSelect(deviceIds: string[])` callback
- Shows platform icon (use text "iOS"/"Android" badge, not Lucide) and device state
- Apply optional `filter` prop to hide unsupported devices (e.g. erase is iOS-only)
- Use existing glass styling: `cmdk-item-text` class pattern, matching existing CommandPalette item look
- Keyboard-navigable via cmdk's built-in Command.Item keyboard handling

**StepRenderer.tsx** — Orchestrates the step flow:
- Props: `action: MultiStepAction`, `onComplete: () => void`, `onBack: () => void`
- Maintains `currentStepIndex` state and `context: StepContext` accumulator
- Renders a breadcrumb/back bar at top: back arrow button + step labels showing progress (e.g. "Screenshot > Select Device")
- Based on `currentStep.type`, renders the appropriate component:
  - `device-select` → `<DevicePicker>` 
  - `confirm` → Confirmation message with "Confirm" and "Cancel" buttons styled as glass-button-primary and glass-button-destructive
  - `execute` → Calls `action.execute(context)`, shows brief loading spinner, then closes
- On step completion, advances `currentStepIndex` and merges result into context
- Back button goes to previous step (or back to action list if on first step)
- Use Tailwind transitions for step content fade (opacity + translateX, 150ms)

Important: All step components render INSIDE the `Command.List` area of the existing cmdk dialog — they replace the action/module list, not overlay it. The cmdk `Command.Input` at top becomes the search/filter for whatever the current step shows (e.g. filtering devices by name in DevicePicker).
