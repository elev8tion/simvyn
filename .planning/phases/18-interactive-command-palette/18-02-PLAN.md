**LocationPicker.tsx** — Location selection with bookmarks and geocoding:
- Two sections rendered as `Command.Group`:
  1. **Bookmarks** group — fetches saved locations from `/api/modules/location/favorites/locations` on mount. Each bookmark as a Command.Item showing name + lat/lon in muted text. Show MapPin icon.
  2. **Search Results** group — appears when user types in the Command.Input. Debounce 300ms, then fetch `/api/modules/location/search?q=${query}&limit=8`. Show results as Command.Items with `displayName` + lat/lon. Show Search icon.
- When no search text and no bookmarks: show empty state "Type to search for a location or save bookmarks in the Location module"
- When searching: show a loading indicator (subtle "Searching..." text) while fetch is pending
- `onSelect(location: { lat: number, lon: number, name: string })` callback
- Use AbortController to cancel in-flight geocoding requests when search text changes
- Handle the `search` value from cmdk — the LocationPicker needs to intercept the Command.Input value. Since cmdk Command.Input is in the parent, pass `search` string as a prop from StepRenderer (which gets it from CommandPalette's search state).

**StepRenderer.tsx** — Add rendering for new step types:

- `location-select` → `<LocationPicker search={search} onSelect={...} />`
- Pass the `search` prop through from CommandPalette to StepRenderer to LocationPicker
- When locale/location selected, store in `context.params` (e.g. `params.locale = "ja_JP"` or `params.location = { lat, lon, name }`) and advance to next step
