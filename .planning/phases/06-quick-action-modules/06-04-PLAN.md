  <action>
    **screenshot-store.ts** — Zustand store:
    - State: `{ captures: CaptureEntry[], isRecording: Record<string, boolean>, loading: boolean }`
    - CaptureEntry: `{ type: "screenshot"|"recording", filename, timestamp, deviceId, deviceName, duration? }`
    - Actions:
      - `captureScreenshot(deviceId: string)` — POST to `/api/modules/screenshot/capture/${deviceId}`, add to captures, show success toast via sonner.
      - `startRecording(deviceId: string)` — POST to `/api/modules/screenshot/record/start/${deviceId}`, set isRecording[deviceId] = true.
      - `stopRecording(deviceId: string)` — POST to `/api/modules/screenshot/record/stop/${deviceId}`, set isRecording[deviceId] = false, add to captures.
      - `fetchHistory()` — GET `/api/modules/screenshot/history`, populate captures.
      - `downloadFile(filename: string)` — Open `/api/modules/screenshot/download/${filename}` in new tab or trigger download.
      - `copyToClipboard(filename: string)` — Fetch the screenshot as blob from download URL, use `navigator.clipboard.write()` with ClipboardItem to copy PNG to host clipboard. Show toast on success/failure. Note: clipboard.write requires HTTPS or localhost — which we have.

    **ScreenshotPanel.tsx** — Dashboard panel following AppPanel/LogPanel patterns:
    - Header row: "Screenshots & Recording" title + device selector dropdown (same pattern as AppPanel).
    - Action bar: "Capture Screenshot" button (Camera icon), "Start Recording" / "Stop Recording" toggle button (Video icon, changes color/label when recording), recording duration timer when active.
    - History grid: CSS grid of capture cards (3 columns on lg, 2 on md, 1 on sm). Each card is a `.glass-panel` with:
      - Thumbnail (for screenshots: `<img>` loading from download URL; for recordings: video icon placeholder with duration)
      - Filename, timestamp, device name
      - Actions: Download button, Copy to clipboard button (screenshots only)
    - Empty state: "No captures yet — take a screenshot or start recording" in glass panel.
    - Register panel: `registerPanel("screenshot", ScreenshotPanel)` at module level.
    - Subscribe to WS "screenshot" channel for real-time recording state updates (useWsListener for "recording-started" and "recording-stopped").

    Follow established visual patterns:
    - `text-base font-medium text-text-primary` for headings
    - `glass-panel` for card containers
    - `bg-bg-surface/60 border border-border` for inputs/selects
    - `rounded-[var(--radius-button)]` for buttons
    - Sonner toasts for async feedback
    - Lucide icons (Camera, Video, Download, Copy, Square for stop)
  </action>
  <verify>TypeScript compiles: `npx tsc --noEmit -p packages/dashboard/tsconfig.json`. Panel renders without errors when screenshot module is selected in sidebar.</verify>
