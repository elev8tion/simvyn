    **Create `packages/modules/push/` module:**

    **package.json**: name `@simvyn/module-push`, type module, dependencies on `@simvyn/types`, `@simvyn/core`, `@simvyn/server`.

    **tsconfig.json**: Same pattern as other modules.

    **templates.ts** — Built-in push notification payload templates:
    Export an array of template objects: `{ id: string, name: string, description: string, payload: object }`.
    Include these templates:
    1. "basic" — Simple alert: `{ aps: { alert: { title: "Title", body: "Body" } } }`
    2. "badge" — Badge update: `{ aps: { badge: 1 } }`
    3. "sound" — Alert with sound: `{ aps: { alert: { title: "Title", body: "Body" }, sound: "default" } }`
    4. "silent" — Silent/content-available: `{ aps: { "content-available": 1 } }`
    5. "rich" — Rich notification with subtitle, category, and custom data: `{ aps: { alert: { title: "Title", subtitle: "Subtitle", body: "Body" }, category: "MESSAGE", "mutable-content": 1 }, customKey: "customValue" }`
    6. "action" — With action buttons: `{ aps: { alert: { title: "Title", body: "Body" }, category: "ACTION_CATEGORY" } }`

    **routes.ts** — Fastify routes (prefix `/api/modules/push`):
    - `POST /send` — Body: `{ deviceId, bundleId, payload: object }`. Validate device is iOS (push via simctl is iOS-only). Write payload to temp JSON file, call `execFileAsync("xcrun", ["simctl", "push", deviceId, bundleId, tempFilePath])`, cleanup temp file. Return `{ success: true }`. Save to send history.
    - `GET /templates` — Return built-in templates from templates.ts.
    - `GET /payloads` — Return saved user payloads from storage. Each: `{ id, name, bundleId?, payload, createdAt }`.
    - `POST /payloads` — Body: `{ name, bundleId?, payload }`. Save payload with generated UUID. Return created payload.
    - `DELETE /payloads/:id` — Delete saved payload by id. Return `{ success: true }`.
    - `GET /history` — Return recent push sends (last 50). Each: `{ bundleId, payload, deviceId, deviceName, timestamp }`.

    Use `createModuleStorage("push")` for persistence (keys: "payloads", "history").

    For the temp file approach: use `mkdtemp` + `writeFile` to create temp JSON, pass to simctl, then `rm` the temp dir in a finally block. Same pattern as app-management's file upload handling.

    **manifest.ts** — Module manifest:
    - name: "push", icon: "bell", capabilities: ["push"]
    - register: register routes (no WS handler needed)
    - cli: `simvyn push <device> --bundle <bundle-id> --payload <json-string>` — Send push notification. Also support `--file <path>` to read payload from a JSON file. Parse JSON string or read file, write to temp file, call simctl push. Print confirmation. Only works on iOS devices — print error and exit(1) if Android device selected.

    **Important notes:**
    - Push notifications via simctl only work on iOS simulators. The module should clearly communicate this limitation — route returns 400 with "Push notifications are only supported on iOS simulators" for Android devices.
    - No adapter method needed — this is simctl-specific, not a cross-platform adapter operation. Call execFileAsync directly in routes and CLI.
    - Use `import { execFile } from "node:child_process"` + `promisify` pattern (not spawn) since simctl push is a one-shot command.
