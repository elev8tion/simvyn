  <name>Task 1: Add tool-settings API endpoints and inject into module registry</name>
  <files>
    packages/server/src/app.ts
  </files>
  <action>
1. Add tool-settings API routes in `app.ts` (after the `/api/health` route):

   a. `GET /api/tool-settings/config` — Returns current config:
      - Read config from `createModuleStorage("tool-settings")` key "config"
      - Return `{ port: number, autoOpen: boolean }` (defaults: port=3847, autoOpen=true)
      - Import `createModuleStorage` from `@simvyn/core`

   b. `PUT /api/tool-settings/config` — Save config:
      - Accept `{ port?: number, autoOpen?: boolean }` in body
      - Merge with existing config
      - Write to storage
      - Return updated config
      - NOTE: This saves the config for NEXT launch — it doesn't restart the server. Show a note in the UI.

   c. `GET /api/tool-settings/storage` — Returns storage usage:
      - Import `getSimvynDir` from `@simvyn/core`
      - Recursively calculate total size of `~/.simvyn/` directory
      - Use a simple recursive `readdir` + `stat` to sum file sizes
      - Return `{ totalBytes: number, humanReadable: string }` (format as "X.X MB" or "X.X KB")

   d. `DELETE /api/tool-settings/data` — Wipe all saved data:
      - Import `rm` from `node:fs/promises`
      - Delete the entire `~/.simvyn/` directory: `rm(getSimvynDir(), { recursive: true, force: true })`
      - Recreate it empty: `mkdir(getSimvynDir(), { recursive: true })`
      - Return `{ wiped: true }`

2. Inject "tool-settings" into module registry:
   After the module loader has run and the modules route is registered, the module list comes from `moduleRegistry`. To add tool-settings to the sidebar without creating a full module, inject it into the registry in `app.ts`:

   After `await fastify.register(moduleLoaderPlugin, ...)`:
   ```ts
   fastify.moduleRegistry.set("tool-settings", {
     name: "tool-settings",
     version: "1.0.0",
     description: "Configure simvyn server and manage stored data",
   });
   ```

   This makes it appear in `GET /api/modules` automatically.

3. Import needed functions at top of app.ts:
   - `import { createModuleStorage, getSimvynDir } from "@simvyn/core";`
   - `import { mkdir, readdir, rm, stat } from "node:fs/promises";`
   - `import { join } from "node:path";`
