Key WS adaptations from sim-location App.tsx:
- sim-location sends flat `{type: "set-location", lat, lon, deviceIds}`. Simvyn sends `{channel: "location", type: "set-location", payload: {lat, lon, deviceId}}`.
- sim-location listens on raw WS `onmessage`. Simvyn uses `useWsListener("location", "playback-position", handler)`.
- sim-location uses multi-device Set. Simvyn passes single `deviceId` from global device store.
- sim-location sends `{type: "list-devices"}` for polling. Simvyn doesn't need this — devices come from global device store via `useDeviceStore`.

Key API path adaptations:
- Reverse geocode: `/api/reverse` → `/api/modules/location/reverse`

The WS message types the server emits (from ws-handler.ts) are: `location-set`, `playback-started`, `playback-position`, `playback-paused`, `playback-resumed`, `playback-stopped`, `playback-complete`, `playback-error`, `speed-changed`. Register listeners for all of these.
