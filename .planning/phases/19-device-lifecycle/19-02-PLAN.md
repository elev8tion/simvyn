  <files>packages/dashboard/src/panels/DevicePanel.tsx</files>
  <action>
Extend the existing DevicePanel with three new capabilities. Keep all existing functionality (boot, shutdown, erase) intact.

**1. Create Simulator Form (top of iOS section)**

Add a "Create Simulator" button in the iOS device section header (next to the "iOS Simulators" heading). Clicking it toggles a collapsible form panel (use glass-panel styling) with:

- Text input for simulator name (placeholder: "My iPhone 17 Pro")
- Dropdown/select for device type — populated from `GET /api/modules/devices/device-types`. Show the `name` field (e.g., "iPhone 17 Pro"). Store the `identifier` for submission. Filter to show only iPhone/iPad types (skip Apple Watch, Apple TV for now — filter by `name.startsWith("iPhone") || name.startsWith("iPad")`).
- Dropdown/select for runtime — populated from `GET /api/modules/devices/runtimes`. Show `name` (e.g., "iOS 26.2"). Store `identifier`. Only show runtimes where `isAvailable === true`.
- "Create" button (glass-button-primary). On click: POST `/api/modules/devices/create` with `{ name, deviceTypeId, runtimeId }`. Show toast on success ("Simulator created"), refresh device list. Disable button during creation, show "Creating..." text.
- "Cancel" button to collapse the form.

Fetch device types and runtimes once when the form first opens (not on mount — lazy). Cache in local state.

**2. Per-Card Actions (iOS device cards only)**

Add three new action buttons to each iOS device card, alongside the existing Boot/Shutdown/Erase buttons:

- **Clone** button (shown always for iOS): Opens a `window.prompt("New name for clone:", device.name + " Copy")`. If name provided, POST `/api/modules/devices/clone` with `{ deviceId, newName }`. Toast on success. Use a copy/duplicate icon from lucide-react (e.g., `Copy` or `CopyPlus`).

- **Rename** button (shown always for iOS): Opens a `window.prompt("New name:", device.name)`. If name provided and different from current, POST `/api/modules/devices/rename` with `{ deviceId, newName }`. Toast on success. Use `Pencil` icon from lucide-react.

- **Delete** button (shown only when state === "shutdown", iOS only): More destructive than erase — this removes the simulator entirely. Use glass-button-destructive. Opens `window.confirm("Delete simulator permanently? This cannot be undone.")`. If confirmed, POST `/api/modules/devices/delete` with `{ deviceId }`. Toast on success. Use `Trash2` icon from lucide-react.

Track `actionInFlight` state for each new action (same pattern as existing boot/shutdown/erase).

**3. Keychain Section (below iOS device grid)**

Add a "SSL Certificates" section below the iOS device grid. Only visible when at least one iOS simulator exists. Uses glass-panel styling:

- Header: "SSL Certificates" with a `ShieldCheck` icon
- Explanation text: "Add root certificates for MITM proxy testing (mitmproxy, Charles Proxy)"
- Device selector dropdown — list all iOS devices (use a simple `<select>` with glass styling, default to first booted device or first device)
- File input: `<input type="file" accept=".pem,.crt,.cer,.der,.p12">` styled with glass-button
- Checkbox: "Trust as root certificate" (default: checked, since root cert is the common use case for proxy testing)
- "Add Certificate" button (glass-button-primary): Read file with FileReader as ArrayBuffer, convert to base64 string, POST `/api/modules/devices/keychain/add-cert` with `{ deviceId, certBase64, isRoot }`. Toast: "Certificate added".
- "Reset Keychain" button (glass-button-destructive): `window.confirm("Reset keychain to defaults?")` → POST `/api/modules/devices/keychain/reset` with `{ deviceId }`. Toast: "Keychain reset".

All emojis forbidden — use Lucide icons only. Follow existing glass-panel, glass-button-primary, glass-button-destructive patterns.

**New file: CreateSimulatorPicker.tsx**

Create a picker component following the LocalePicker pattern (searchable list inside cmdk). This picker handles the full create-simulator flow in one palette step:

Props: `{ onSelect: (params: { name: string; deviceTypeId: string; runtimeId: string; deviceTypeName: string }) => void }`

Internal state: `phase` — cycles through `"name"` → `"deviceType"` → `"runtime"`.

- **Phase "name"**: Show a text input (use Command.Input or a regular input styled consistently). Placeholder: "Simulator name (e.g., My iPhone)". On Enter with non-empty value, store name, advance to "deviceType" phase.

- **Phase "deviceType"**: Fetch `GET /api/modules/devices/device-types` on mount (cache result). Show searchable list of device types using `Command.Item` entries. Filter by search query (match against `name`). Pre-filter to iPhone and iPad types. On select, store deviceTypeId and deviceTypeName, advance to "runtime" phase.

- **Phase "runtime"**: Fetch `GET /api/modules/devices/runtimes` on mount (cache with deviceTypes). Show searchable list of available runtimes using `Command.Item`. Filter by search query. On select, call `onSelect({ name, deviceTypeId, runtimeId, deviceTypeName })`.

Show a breadcrumb/progress indicator at top: "Name → Device Type → Runtime" with current phase highlighted (use muted text for future steps, bright for current).

Style: same glass aesthetic as LocalePicker — transparent backgrounds, text-white, consistent with command palette theme. No emojis.

   - Icon: `Pencil` from lucide-react
   - Steps: `[{ type: "device-select", label: "Select simulator to rename", multi: false, filter: (d) => d.platform === "ios" }, { type: "parameter", label: "New name", placeholder: "New simulator name", paramKey: "newName" }, { type: "confirm", message: (ctx) => \`Rename to "\${ctx.params.newName}"?\` }]`
   - Execute: `POST /api/modules/devices/rename` with `{ deviceId: ctx.selectedDeviceIds[0], newName: ctx.params.newName }`. Toast: "Simulator renamed".
