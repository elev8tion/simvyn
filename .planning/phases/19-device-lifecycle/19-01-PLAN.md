**In packages/types/src/device.ts:**

Add two new interfaces after the existing `AppInfo` type:

```typescript
interface DeviceType {
  identifier: string   // e.g. "com.apple.CoreSimulator.SimDeviceType.iPhone-17-Pro"
  name: string         // e.g. "iPhone 17 Pro"
}

interface SimRuntime {
  identifier: string   // e.g. "com.apple.CoreSimulator.SimRuntime.iOS-26-2"
  name: string         // e.g. "iOS 26.2"
  version: string      // e.g. "26.2"
  isAvailable: boolean
}
```

Add `"deviceLifecycle"` and `"keychain"` to the `PlatformCapability` union type.

Add these optional methods to `PlatformAdapter`:

```typescript
listDeviceTypes?(): Promise<DeviceType[]>
listRuntimes?(): Promise<SimRuntime[]>
createDevice?(name: string, deviceTypeId: string, runtimeId?: string): Promise<string>  // returns new UUID
cloneDevice?(deviceId: string, newName: string): Promise<string>  // returns new UUID
renameDevice?(deviceId: string, newName: string): Promise<void>
deleteDevice?(deviceId: string): Promise<void>
addKeychainCert?(deviceId: string, certData: Buffer, isRoot: boolean): Promise<void>
resetKeychain?(deviceId: string): Promise<void>
```

Export `DeviceType` and `SimRuntime` from the package.

**In packages/core/src/adapters/ios.ts:**

Implement all 8 methods inside the `createIosAdapter()` factory, following the existing `execFileAsync("xcrun", ["simctl", ...])` pattern:

- `listDeviceTypes()`: Run `xcrun simctl list devicetypes --json`. Parse `stdout` → `JSON.parse(stdout).devicetypes`. Map each to `{ identifier, name }`. Filter to entries that have a `name` property (some are internal/legacy).

- `listRuntimes()`: Run `xcrun simctl list runtimes --json`. Parse `stdout` → `JSON.parse(stdout).runtimes`. Map to `{ identifier, name, version, isAvailable }`. Extract version from the `name` field (e.g., "iOS 26.2" → "26.2") or from the `version` property if present.

- `createDevice(name, deviceTypeId, runtimeId?)`: Run `xcrun simctl create <name> <deviceTypeId>` with optional `<runtimeId>` appended. `stdout.trim()` is the new device UUID — return it.

- `cloneDevice(deviceId, newName)`: Run `xcrun simctl clone <deviceId> <newName>`. `stdout.trim()` is the new UUID — return it.

- `renameDevice(deviceId, newName)`: Run `xcrun simctl rename <deviceId> <newName>`.

- `deleteDevice(deviceId)`: Run `xcrun simctl delete <deviceId>`. Guard: the device should be in a shutdown state before deletion (the route layer handles this validation, but the adapter can catch "Invalid device state" errors and re-throw with a clear message).

- `addKeychainCert(deviceId, certData, isRoot)`: Write `certData` to a temp file via `fs.promises.mkdtemp` + `fs.promises.writeFile`. Run `xcrun simctl keychain <deviceId> add-root-cert <tempPath>` if `isRoot`, else `xcrun simctl keychain <deviceId> add-cert <tempPath>`. Clean up temp file in a `finally` block.

- `resetKeychain(deviceId)`: Run `xcrun simctl keychain <deviceId> reset`.

Add `"deviceLifecycle"` and `"keychain"` to the array returned by `capabilities()`.

**In packages/modules/device-management/routes.ts:**

Add 8 new routes to the existing `deviceRoutes` Fastify plugin, following the established pattern (lookup device via `fastify.deviceManager`, get adapter, call method, refresh, return result):

1. `GET /device-types` — Get the iOS adapter from deviceManager. Call `adapter.listDeviceTypes()`. Return `{ deviceTypes }`. If method unavailable, return `{ deviceTypes: [] }`.

2. `GET /runtimes` — Same pattern. Call `adapter.listRuntimes()`. Return `{ runtimes }`. If method unavailable, return `{ runtimes: [] }`.

3. `POST /create` — Body: `{ name: string, deviceTypeId: string, runtimeId?: string }`. Validate `name` is non-empty. Get iOS adapter. Call `adapter.createDevice(name, deviceTypeId, runtimeId)`. Call `await fastify.deviceManager.refresh()`. Return `{ success: true, deviceId: <returned UUID> }`.

4. `POST /clone` — Body: `{ deviceId: string, newName: string }`. Lookup device, validate it exists and is iOS. Get adapter. Call `adapter.cloneDevice(deviceId, newName)`. Refresh. Return `{ success: true, deviceId: <returned UUID> }`.

5. `POST /rename` — Body: `{ deviceId: string, newName: string }`. Lookup device, validate exists and is iOS. Call `adapter.renameDevice(deviceId, newName)`. Refresh. Return `{ success: true }`.

6. `POST /delete` — Body: `{ deviceId: string }`. Lookup device, validate exists, is iOS, AND state === "shutdown" (return 400 if booted: "Device must be shut down before deleting"). Call `adapter.deleteDevice(deviceId)`. Refresh. Return `{ success: true }`.

7. `POST /keychain/add-cert` — Body: `{ deviceId: string, certBase64: string, isRoot: boolean }`. Lookup device, validate iOS. Decode base64: `Buffer.from(certBase64, "base64")`. Call `adapter.addKeychainCert(deviceId, certBuffer, isRoot)`. Return `{ success: true }`.

8. `POST /keychain/reset` — Body: `{ deviceId: string }`. Lookup device, validate iOS. Call `adapter.resetKeychain(deviceId)`. Return `{ success: true }`.

For routes 1-2, to get the iOS adapter without a specific device: use `fastify.deviceManager.getAdapters()` or create the iOS adapter directly — follow the pattern already used in the capabilities endpoint.

Error handling: wrap each route body in try/catch, return `reply.status(500).send({ error: e.message })`.
