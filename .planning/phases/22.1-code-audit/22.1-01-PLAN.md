<task type="auto">
  <name>Task 1: Audit git history for secrets and sensitive data</name>
  <files>.gitignore</files>
  <action>
Run a comprehensive security scan of the entire git history:

1. Search for secret patterns in all diffs:
   - `git log -p --all -S "PRIVATE_KEY"` — private keys
   - `git log -p --all -S "API_KEY"` — API keys
   - `git log -p --all -S "SECRET"` — secrets
   - `git log -p --all -S "password" -- '*.ts' '*.js' '*.json'` — passwords (exclude docs)
   - `git log -p --all -S "token" -- '*.ts' '*.js' '*.json'` — tokens
   - `git log -p --all -S "Bearer "` — auth headers
   - `git log -p --all -S "sk_live" -S "pk_live" -S "sk_test"` — Stripe keys
   - `git log -p --all -S "-----BEGIN"` — PEM/certificate blocks

2. Search for sensitive files ever added to history:
   - `git log --diff-filter=A --name-only --pretty=format:"" -- '*.env*' '*.pem' '*.key' '*.p12' '*.pfx' '*credentials*' '*secret*' '*.keystore'`

3. Check current working tree:
   - `find . -name '.env*' -not -path './node_modules/*'`
   - `find . -name '*.pem' -not -path './node_modules/*'`
   - `find . -name 'credentials*' -not -path './node_modules/*'`
   - Verify no hardcoded URLs with embedded credentials (grep for `://.*:.*@`)

4. Check for accidentally committed large/binary files that shouldn't be public:
   - `git log --diff-filter=A --name-only --pretty=format:"" -- '*.sqlite' '*.db'`
   - Look for any test fixtures or user data in the repo

If any issues found: Document them and clean using `git filter-branch` or BFG Repo Cleaner as needed.
If clean: Confirm in the summary.
  </action>
  <verify>All git history search commands return empty (no matches). No sensitive files in working tree.</verify>
  <done>Complete audit confirms: zero secrets, zero credential files, zero sensitive data in 270 commits of history</done>
</task>
