1. In `packages/types/src/device.ts`:
   - Add `getClipboard?(deviceId: string): Promise<string>;` to the `PlatformAdapter` interface
   - Add `setClipboard?(deviceId: string, text: string): Promise<void>;` to the `PlatformAdapter` interface
   - The `"clipboard"` capability already exists in PlatformCapability — no change needed there

2. In `packages/core/src/adapters/ios.ts`:
   - Add `getClipboard` method: `xcrun simctl pbpaste <deviceId>`
     - `const { stdout } = await execFileAsync("xcrun", ["simctl", "pbpaste", deviceId])`
     - Return `stdout`
   - Add `setClipboard` method: pipe text to `xcrun simctl pbcopy <deviceId>` via stdin
     - Use `spawn("xcrun", ["simctl", "pbcopy", deviceId])` and write text to stdin, then end stdin
     - Wait for process to exit via a Promise wrapper around the `close` event
   - The capability `"clipboard"` is already in the iOS capabilities array — no change needed

3. In `packages/core/src/adapters/android.ts`:
   - Add `getClipboard: undefined` — Android has no reliable way to read clipboard via adb without a helper app. Set to undefined so the API returns 400 for unsupported operation.
   - Add `setClipboard` method:
     - Escape the text for shell: replace special characters. Use `adb -s <deviceId> shell input text <escaped_text>`
     - For the escaping: replace spaces with `%s`, and shell-escape other special chars. Android `input text` treats `%s` as space. Also escape `(`, `)`, `<`, `>`, `|`, `;`, `&`, `*`, `\`, `~`, `"`, `'`, ` ` with backslash.
     - Alternative for longer/complex text: use `adb -s <deviceId> shell am broadcast -a clipper.set -e text "<text>"` — but this requires a helper app. Better approach: write text to a temp file on device, then use `adb shell input text` for simple cases.
     - For practical reliability, use `input text` for texts under 256 chars. For longer texts, fall back to writing to a temp file and using `adb shell cmd clipboard set-text` (available on Android 12+). Wrap the clipboard service call in try/catch and fall back to `input text` truncated to 256 chars with a warning.
   - Add `"clipboard"` to the Android capabilities array (it's currently missing — only iOS has it)
   - Guard against `avd:` prefix device IDs

7. Create `packages/modules/clipboard/routes.ts`:
   Export `async function clipboardRoutes(fastify: FastifyInstance)`:

   **GET /get/:deviceId** — Read device clipboard
   - Check device exists and is booted (404/400)
   - Get adapter, check `adapter.getClipboard` exists (return 400 with `{ error: "Clipboard read not supported for this platform" }` if not)
   - Call `adapter.getClipboard(device.id)`
   - Return `{ text: string }`

   **POST /set/:deviceId** — Write to device clipboard
   - Body: `{ text: string }`
   - Check device exists and is booted (404/400)
   - Get adapter, check `adapter.setClipboard` exists (400 if not)
   - Call `adapter.setClipboard(device.id, text)`
   - Return `{ success: true }`

   Follow same error handling pattern as settings/routes.ts.

8. Create `packages/modules/clipboard/cli.ts`:
   Export `function registerClipboardCli(program: Command)`:
   Create a `clipboard` command group with subcommands:

   **`simvyn clipboard get <device>`**
   - Lazy-import `@simvyn/core`
   - Create adapters + device manager, refresh, find device
   - Check `adapter.getClipboard` exists (exit 1 with "Clipboard read not supported for <platform>")
   - Call `adapter.getClipboard(target.id)`
   - Print the clipboard text to stdout (just the text, no prefix — enables piping)
   - `dm.stop()` in finally

   **`simvyn clipboard set <device> <text>`**
   - Lazy-import `@simvyn/core`
   - Create adapters + device manager, refresh, find device
   - Check `adapter.setClipboard` exists
   - Call `adapter.setClipboard(target.id, text)`
   - Print `Clipboard set on <device.name>`
   - `dm.stop()` in finally

9. Create `packages/modules/clipboard/manifest.ts`:

   - register() registers clipboardRoutes
   - cli() delegates to registerClipboardCli
   - capabilities: `["clipboard"]`
