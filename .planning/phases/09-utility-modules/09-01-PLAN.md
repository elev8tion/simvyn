  <action>
1. In `packages/types/src/device.ts`:
   - Add `"crashLogs"` to the `PlatformCapability` union type
   - Add a `CrashLogEntry` interface:
     ```
     export interface CrashLogEntry {
       id: string;
       process: string;
       timestamp: string;
       path?: string;
       preview: string;
     }
     ```

2. Create `packages/modules/crash-logs/ios-crashes.ts`:
   - Export `async function listIosCrashLogs(opts?: { app?: string; since?: string }): Promise<CrashLogEntry[]>`
   - Read `~/Library/Logs/DiagnosticReports/` directory using `fs.readdir`
   - Also check `~/Library/Logs/DiagnosticReports/Retired/` if it exists
   - Filter files with extensions `.ips`, `.crash`, `.ips.ca` (compressed crash logs)
   - For each file:
     - Parse the filename to extract process name (format: `ProcessName-YYYY-MM-DD-HHMMSS.ips`)
     - Use `fs.stat` to get modification time as timestamp
     - Read first 500 chars as preview
     - Generate id from filename (use path basename without extension)
   - If `opts.app` provided, filter entries where process matches (case-insensitive includes)
   - If `opts.since` provided, filter entries where timestamp is after the given ISO date string
   - Sort by timestamp descending (newest first)
   - Return array of CrashLogEntry
   - Export `async function readIosCrashLog(logId: string): Promise<string>`
   - Find the file in DiagnosticReports matching the id (basename), read and return full content
   - Use `os.homedir()` to resolve ~ paths

3. Create `packages/modules/crash-logs/android-crashes.ts`:
   - Export `async function listAndroidCrashLogs(deviceId: string, opts?: { app?: string; since?: string }): Promise<CrashLogEntry[]>`
   - Guard against `avd:` prefix device IDs (same pattern as other Android adapter methods — return empty array)
   - Run `adb -s <deviceId> logcat -d *:E` via `execFileAsync` to dump error-level logs
   - Parse output line by line. Android logcat default format: `MM-DD HH:MM:SS.mmm PID TID LEVEL TAG: message`
   - Group consecutive lines by PID+TAG into crash entries
   - For each entry:
     - id: generate from line index or hash of first line
     - process: TAG field
     - timestamp: parse the MM-DD HH:MM:SS into ISO format (use current year since logcat doesn't include year)
     - preview: first 200 chars of the grouped message
   - If `opts.app` provided, filter entries where process/TAG matches (case-insensitive includes)
   - If `opts.since` provided, filter by parsed timestamp
   - Sort by timestamp descending
   - Also try `adb -s <deviceId> shell dumpsys dropbox --print` to get tombstone entries. This command may fail on some devices — wrap in try/catch and merge results if available. Parse tombstone output for `SYSTEM_TOMBSTONE` tag entries.
   - Return merged + deduplicated array of CrashLogEntry
   - Use `promisify(execFile)` from node:util for all adb commands
  </action>

4. Create `packages/modules/crash-logs/routes.ts`:
   Export `async function crashLogRoutes(fastify: FastifyInstance)` with these endpoints:

   **GET /list/:deviceId** — List crash logs for a device
   - Query params: `app` (optional string), `since` (optional ISO date string)
   - Check device exists and is booted (same 404/400 pattern as other modules)
   - If `device.platform === "ios"`, call `listIosCrashLogs({ app, since })`
   - If `device.platform === "android"`, call `listAndroidCrashLogs(device.id, { app, since })`
   - Return `{ logs: CrashLogEntry[] }`

   **GET /view/:deviceId/:logId** — View a specific crash log
   - For iOS: call `readIosCrashLog(logId)` and return `{ content: string }`
   - For Android: the full content is already in the logcat dump — re-run `adb logcat -d *:E` and extract the matching entry's full text. OR, if the logId references a tombstone, attempt `adb shell dumpsys dropbox --print <tag>`. Return `{ content: string }`
   - Return 404 if log not found

   Follow the same error handling pattern as settings/routes.ts.

5. Create `packages/modules/crash-logs/cli.ts`:
   Export `function registerCrashLogsCli(program: Command)`:

   - Lazy-import `@simvyn/core` inside action handler
   - Create adapters + device manager, refresh, find device by ID/prefix
   - If `--view` provided:
     - For iOS: call `readIosCrashLog(logId)`, print full content
     - For Android: re-query logcat and print matching entry
   - Otherwise: list crash logs with platform-appropriate function
   - Print table: `Timestamp  Process  Preview`
   - Always call `dm.stop()` in finally block
