  <action>
    **1. Create search store** at `packages/dashboard/src/panels/logs/stores/search-store.ts`:

    Zustand store with state:
    - `isOpen: boolean` (default false)
    - `query: string` (default "")
    - `isRegex: boolean` (default false)
    - `matchIndices: number[]` — indices of matching entries in the filtered+reversed display array
    - `currentMatchIdx: number` — position within matchIndices (0-based), -1 when no matches
    - `totalMatches: number` — matchIndices.length

    Actions:
    - `open()` — sets isOpen: true, focuses input
    - `close()` — sets isOpen: false, clears query, resets matchIndices/currentMatchIdx
    - `setQuery(q: string)` — sets query, resets currentMatchIdx to 0 (or -1 if no matches)
    - `toggleRegex()` — toggles isRegex
    - `setMatches(indices: number[])` — sets matchIndices and totalMatches, clamps currentMatchIdx
    - `nextMatch()` — increments currentMatchIdx, wraps around to 0
    - `prevMatch()` — decrements currentMatchIdx, wraps around to last

    Export `useSearchStore`.

    **2. Create SearchOverlay component** at `packages/dashboard/src/panels/logs/SearchOverlay.tsx`:

    Floating overlay positioned absolute top-right of the log content area (inside the flex container, above LogList). Use `glass-panel` styling with compact padding. Layout:

    ```
    [ Search input ][ Aa/.*  ][ ▲ ][ ▼ ][ 3 of 47 ][ ✕ ]
    ```

    - Text input: `glass-input` class, placeholder "Search logs...", auto-focused on open. Controlled by search store query. Debounce query updates by 100ms.
    - Regex toggle button: small glass-button, shows ".*" text. Active state with accent border when isRegex is true.
    - Prev/next buttons: small glass-buttons with ChevronUp/ChevronDown icons from lucide-react. Disabled when totalMatches === 0.
    - Match counter: text showing `{currentMatchIdx + 1} of {totalMatches}` when matches > 0, "No results" when query is non-empty but matches === 0, hidden when query is empty.
    - Close button: X icon (lucide), calls close().

    Keyboard handling within the overlay:
    - Enter → nextMatch()
    - Shift+Enter → prevMatch()
    - Escape → close()

    Animation: use Framer Motion AnimatePresence with slide-in from top-right (y: -10, opacity: 0 → y: 0, opacity: 1). Exit reverses. Keep it subtle.

    **3. Wire Cmd+F in LogPanel.tsx:**

    Add a `useEffect` in LogPanel that registers a `window.addEventListener("keydown", ...)` handler:
    - Check `(e.metaKey || e.ctrlKey) && e.key === "f"`
    - Call `e.preventDefault()` to suppress browser find
    - Call `useSearchStore.getState().open()`
    - Only active when LogPanel is mounted (the effect cleanup removes the listener)

    Render `<SearchOverlay />` inside LogPanel, positioned absolute within the log content area (after LogToolbar, overlaying the top-right of the LogList area). Wrap the LogList + SearchOverlay in a `relative` container div.

    On close (when search store `isOpen` transitions false), remove highlights by clearing state — LogList will stop highlighting automatically.

    **1. Add text highlighting to LogList.tsx:**

    Create a `highlightText(text: string, query: string, isRegex: boolean): ReactNode` helper function (can be defined in LogList.tsx or a small utility):
    - If query is empty, return text as-is
    - Build a regex from query: if isRegex, use `new RegExp(query, "gi")` with try/catch for invalid regex; if plain text, use `new RegExp(escapeRegex(query), "gi")`
    - Use `String.split(regex)` + regex match to interleave plain text and highlighted spans
    - Highlighted spans use a subtle style: `<mark>` element with custom class or inline style — subtle amber/gold background with low opacity (e.g., `bg-amber-500/20 text-amber-200 rounded-sm px-0.5`), NOT harsh yellow
    - The currently focused match (the one at currentMatchIdx) gets a slightly stronger style (e.g., `bg-amber-500/40 ring-1 ring-amber-500/50`) to distinguish from other matches. Use a `data-match-index` attribute to identify.

    In the `itemContent` render of LogList's Virtuoso:
    - Read `query`, `isRegex`, `currentMatchIdx`, `matchIndices` from useSearchStore
    - Apply `highlightText()` to the message span (and optionally processName span)
    - Only apply highlighting when query is non-empty

    **2. Compute match indices:**

    In LogList.tsx, add a `useEffect` that recomputes matches whenever `query`, `isRegex`, or the displayed `entries` array changes:
    - Iterate over the displayed (filtered+reversed) entries array
    - For each entry, test if `entry.message` (or `entry.processName`) matches the search query
    - Collect indices of matching entries
    - Call `useSearchStore.getState().setMatches(matchingIndices)`
    - Debounce or use requestAnimationFrame to avoid blocking on large datasets

    **3. Scroll-to-match when navigating:**

    Add a `useEffect` that watches `currentMatchIdx` and `matchIndices` from the search store:
    - When currentMatchIdx changes and is valid (>= 0):
      - Get the entry index from `matchIndices[currentMatchIdx]`
      - Call `virtuosoRef.current?.scrollToIndex({ index: entryIndex, align: "center", behavior: "auto" })` — use "center" for now (simpler than "just enough" with virtuoso). The `behavior: "auto"` avoids smooth-scroll jank in large lists.

    **4. Add three-dot overflow menu to LogToolbar.tsx:**

    Add a `MoreVertical` (or `Ellipsis`) icon button from lucide-react at the end of the toolbar (after the entry count span).

    On click, show a simple dropdown menu positioned below the button:
    - Use a local `useState` for open/close, render conditionally
    - Position with `absolute right-0 top-full mt-1`
    - Style with `glass-panel` class, compact padding, z-50
    - Single menu item: "Search" with `⌘F` shortcut hint on the right
    - Clicking "Search" calls `useSearchStore.getState().open()` and closes the dropdown
    - Click outside closes the dropdown (use a `useEffect` with document click listener)

    The dropdown should feel minimal — a simple glass-panel popup, not a heavy component library menu.
