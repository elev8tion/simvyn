# Phase 30.2: Log Search - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Search for specific log entries and see surrounding context — distinct from the existing filter which hides non-matching entries. Filter removes non-matches from view; search locates matches within the full stream, highlights them, and lets you navigate between hits. GitHub issue #4.

</domain>

<decisions>
## Implementation Decisions

### Search result presentation
- All logs stay visible — matching entries get highlighted in-place (no collapsing or hiding)
- Matched text emphasis should be elegant and subtle — Claude's discretion on exact visual treatment (background highlight, bold+color, etc.)
- Plain text matching by default, with optional regex support (toggle or /regex/ syntax)

### Search navigation
- Both keyboard shortcuts AND visible buttons — standard IDE/terminal pattern
- Enter for next match, Shift+Enter for previous, plus small arrow buttons in the search bar
- Match counter with position: "3 of 47" showing current match index and total count
- Navigating to a match scrolls just enough to bring it into viewport if not already visible
- Cmd+F / Ctrl+F intercepted when Logs panel is active — opens the log search bar directly

### Search vs filter coexistence
- Search and filter are independent — both can be active simultaneously
- Filter narrows visible entries, search finds within the filtered results
- Search bar appears as a floating overlay (VS Code-style, slides in from top-right of log area)
- Dismissed via Escape key or X button — clears highlights on dismiss
- Three-dot overflow menu in the log toolbar includes a "Search" option for discoverability (in addition to Cmd+F)

### Claude's Discretion
- Exact highlight/focus visual treatment (elegant and subtle)
- Whether active/focused match looks different from other matches
- CLI search support — dashboard-only or add --search flag to `simvyn logs`
- Search bar animation/transition details

</decisions>

<specifics>
## Specific Ideas

- Should feel like VS Code's Cmd+F or terminal find — familiar IDE-like pattern
- The three-dot menu entry increases feature discoverability for users who don't know Cmd+F
- Search operates on whatever entries are currently loaded/visible (client-side on the filtered + paginated buffer)

</specifics>
