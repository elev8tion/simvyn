  <action>
Create `packages/dashboard/src/panels/collections/ApplyModal.tsx` — a modal overlay for applying a collection to selected devices.

**Props:** `{ collection: Collection; actions: SerializedAction[]; open: boolean; onClose: () => void }`

**Three phases of the modal:**

**Phase 1 — Device Selection + Compatibility Summary:**
1. Render a fixed overlay (z-50, bg-black/50 backdrop, centered glass-panel, max-w-lg w-full, max-h-[80vh] overflow-y-auto).
2. Header: Collection name (text-base font-medium), close button (X icon, top-right).
3. Device list: Read devices from `useDeviceStore(s => s.devices)`. Show only booted devices. Each device is a selectable row (checkbox + device name + platform badge). Use local `selectedDeviceIds: Set<string>` state.
4. **Compatibility summary** (below device list): For each selected device, compute how many steps will be skipped based on platform incompatibility. Use the same iOS-only action set: `set-status-bar, clear-status-bar, set-increase-contrast, set-content-size`. For an Android device, count steps using iOS-only actions. Display: `"Device Name: N of M steps will run (K skipped)"` per selected device, in text-xs text-text-muted. If any device has skips, show a small yellow warning icon.
5. Footer: "Apply" button (glass-button-primary) disabled when no devices selected. Shows `⌘↵` hint. Wire up Cmd+Enter keyboard shortcut (capture phase event listener) to trigger apply when devices are selected.

**Phase 2 — Live Execution (status matrix):**
1. When user clicks Apply or presses Cmd+Enter: POST `/api/modules/collections/${collection.id}/execute` with `{ deviceIds: Array.from(selectedDeviceIds) }`. Store the returned `runId`. Switch modal to execution phase.
2. Subscribe to WS events via `useWsListener("collections", "step-progress", handler)` and `useWsListener("collections", "run-completed", handler)`.
3. Render a **status matrix table**: rows = steps (from collection.steps, resolved to action labels via the actions prop), columns = selected devices. Each cell shows status:
   - `pending`: dim dash (text-text-muted "—")
   - `running`: spinning Loader2 icon (animate-spin, text-accent-blue, 14px)
   - `success`: Check icon (text-green-400, 14px)
   - `failed`: X icon (text-red-400, 14px) with tooltip showing error
   - `skipped`: SkipForward icon (text-yellow-400, 14px)
4. Track execution state locally: `executionRun: ExecutionRun | null`. On `step-progress` WS event, update `executionRun` if `runId` matches. On `run-completed`, mark as done and show completion summary.
5. The step-progress payload IS the full ExecutionRun object (same shape as types). Just set the state directly.

**Phase 3 — Completion:**
1. When run completes, show summary below the matrix: total success/failed/skipped counts in colored text.
2. Show "Done" button (glass-button-primary) that calls onClose.
3. Also show "Run Again" button (glass-button) that resets to Phase 1 with same device selection.

Clicking it opens the ApplyModal for that collection. Add state: `applyingCollection: Collection | null`. Render `<ApplyModal collection={applyingCollection} actions={actions} open={!!applyingCollection} onClose={() => setApplyingCollection(null)} />` where `actions` comes from `useCollectionsStore(s => s.actions)`.

  <action>
Update `packages/dashboard/src/components/command-palette/actions.tsx` to dynamically include saved collections as command palette actions.

**Approach:** The `getActions` function currently returns a static array. Change it to accept an optional `collections` parameter: `getActions(navigate: NavigateFunction, collections?: Array<{ id: string; name: string; steps: { actionId: string }[] }>)`.

For each collection, generate a `MultiStepAction`:
```typescript
{
  id: `collection:${collection.id}`,
  label: `Apply: ${collection.name}`,
  description: `Run ${collection.steps.length} step collection`,
  icon: <Layers size={18} />,  // import Layers from lucide-react
  steps: [
    { id: "pick-devices", type: "device-select", label: "Select Devices", multi: true }
  ],
  execute: async (ctx) => {
    const res = await fetch(`/api/modules/collections/${collection.id}/execute`, {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ deviceIds: ctx.selectedDeviceIds }),
    });
    if (res.ok) {
      toast.success(`Applying "${collection.name}"...`);
    } else {
      const data = await res.json().catch(() => ({ error: "Apply failed" }));
      toast.error(data.error || "Apply failed");
    }
  },
}
```

Append these collection actions after the existing static actions array (before the return). This way collections appear in the "Actions" group of the command palette.

**In CommandPalette.tsx:** Update the `useMemo` for `actions` to pass collections. Read collections from the collections store:
```typescript
import { useCollectionsStore } from "../panels/collections/stores/collections-store";
// Inside component:
const collections = useCollectionsStore(s => s.collections);
const actions = useMemo(() => getActions(navigate, collections), [navigate, collections]);
```

Also ensure collections are loaded: Add a `useEffect` that calls `fetchCollections` on mount if collections is empty (the store's fetchCollections is idempotent — it just GETs from the API).

Import `Layers` from lucide-react in actions.tsx.
