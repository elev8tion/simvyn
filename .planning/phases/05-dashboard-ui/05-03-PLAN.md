Update `ModuleShell.tsx` to add spring-based animations when switching modules. The current pattern uses `display: none/block` to toggle visibility — this preserves module state. Keep this pattern but layer Framer Motion `motion.div` for a subtle entrance animation.

Strategy: The display:none/block toggle stays for state persistence. When a module becomes active (display changes from none to block), we want a subtle fade+slide-up animation. Since display:none elements can't animate, use a wrapper approach:

- Keep the display:none/block outer div for state persistence
- Inside each module's container, wrap the Panel component in a `motion.div` that only animates when it becomes the active module
- Track the `activeModule` and use `AnimatePresence` or a `key` change to trigger entrance animation

Actually, the simplest approach that preserves state: use `motion.div` with `animate` based on `isActive`. When active, animate opacity 0→1 and y 8→0. When inactive, the parent has `display:none` so the animation state doesn't matter.

```tsx
import { motion } from "framer-motion";
// ... existing imports

const panelSpring = {
  type: "spring" as const,
  stiffness: 400,
  damping: 30,
  mass: 0.8,
};

// Inside the map, wrap the Panel:
<div
  key={mod.name}
  className="absolute inset-0 overflow-auto"
  style={{ display: isActive ? "block" : "none" }}
>
  {Panel ? (
    <Suspense fallback={<LoadingSkeleton />}>
      <motion.div
        key={`panel-${mod.name}`}
        initial={{ opacity: 0, y: 8 }}
        animate={{ opacity: 1, y: 0 }}
        transition={panelSpring}
      >
        <Panel />
      </motion.div>
    </Suspense>
  ) : (
    // ... no panel state unchanged
  )}
</div>
```

The `initial` + `animate` on `motion.div` will play the entrance animation once when the component first mounts. Since we use display:none/block (not mount/unmount), subsequent switches won't re-trigger. This is fine — the animation plays on first load of each panel, which is the moment that matters most.

If we want animation on every switch, we can add a `key` that includes the active module state. But this would remount the component and lose state. So we keep the current behavior: animate once on mount, persist state across switches.

Alternative for re-animation on switch: use `useEffect` + animation controls. When `isActive` changes to true, trigger animation via controls.animate(). This gives us animation on every switch without remounting:

```tsx
import { motion, useAnimationControls } from "framer-motion";

// Inside the map, for each module:
function AnimatedPanel({ isActive, children }: { isActive: boolean; children: React.ReactNode }) {
  const controls = useAnimationControls();
  
  useEffect(() => {
    if (isActive) {
      controls.set({ opacity: 0, y: 8 });
      controls.start({ opacity: 1, y: 0 }, panelSpring);
    }
  }, [isActive, controls]);

  return (
    <motion.div animate={controls}>
      {children}
    </motion.div>
  );
}
```

Use this `AnimatedPanel` wrapper approach. It re-animates every time the module becomes active, without remounting.
