1. **Server routes** (`packages/modules/screenshot/routes.ts`):

   Add two new endpoints:

   a. `DELETE /history/:filename` — Delete individual capture:
      - Read history from storage
      - Find entry by filename
      - If not found, return 404
      - Delete the file from disk (`unlink` the file at capturesDir or recordingsDir)
      - Remove entry from history array
      - Write updated history back to storage
      - Return 200 `{ deleted: true }`

   b. `DELETE /history` — Clear all captures:
      - Read history from storage
      - For each entry, delete the file from disk (try/catch each — some files may already be gone)
      - Write empty array to history storage
      - Return 200 `{ cleared: true, count: N }`

   Import `unlink` from `node:fs/promises`.

2. **Screenshot store** (`screenshot-store.ts`):

   Add two new actions:

   a. `deleteCapture: (filename: string) => Promise<void>`:
      - `DELETE /api/modules/screenshot/history/${filename}`
      - On success: remove from `captures` array in state, toast success
      - On error: toast error

   b. `clearAllCaptures: () => Promise<void>`:
      - `DELETE /api/modules/screenshot/history`
      - On success: set `captures` to `[]`, toast success
      - On error: toast error

3. **ScreenshotPanel UI** (`ScreenshotPanel.tsx`):

   a. Add delete button to `CaptureCard`:
      - Add a red/destructive trash icon button alongside the download/copy buttons

   b. Add "Clear All" button to the history section header:
      - When captures exist, show a "Clear All" button (glass-button-destructive) in the header area
      - `onClick` calls `clearAllCaptures()` from the store
      - Show a simple `window.confirm("Delete all captures?")` before proceeding

   c. Wire new store actions:
      - Destructure `deleteCapture` and `clearAllCaptures` from `useScreenshotStore()`
      - Pass `onDelete` prop to CaptureCard

4. Build verification: `npm run build`
