**sqlite-inspector.ts** — SQLite operations via better-sqlite3:
- `openReadonly(dbPath: string): Database.Database`: Open with `{ readonly: true, timeout: 10000 }`. Return the Database instance. Caller is responsible for closing.
- `getTables(db: Database.Database)`: Query `SELECT name FROM sqlite_master WHERE type='table' AND name NOT LIKE 'sqlite_%' ORDER BY name`. For each table, get columns via `PRAGMA table_info("<name>")` and row count via `SELECT count(*) FROM "<name>"`. Return array of `{ name, columns: Array<{cid, name, type, notnull, dflt_value, pk}>, rowCount }`.
- `queryTable(db, table, { limit, offset, orderBy?, orderDir? })`: Build query `SELECT * FROM "<table>" ORDER BY "<orderBy>" <orderDir> LIMIT ? OFFSET ?`. Use `.all(limit, offset)` for rows, `.columns()` for column metadata. Return `{ rows, columns }`. Always double-quote table and column names.
- `runQuery(db, sql)`: Prepare the statement. If `stmt.reader` is true (SELECT), return `{ type: "rows", rows: stmt.all(), columns: stmt.columns() }`. Otherwise (INSERT/UPDATE/DELETE), return `{ type: "run", changes: stmt.run().changes }`. Wrap in try/catch — rethrow SqliteError with the original message for the route to handle.
- `findDatabases(dirPath: string)`: Recursively walk dirPath using `readdir({ withFileTypes: true })`. Collect files with extensions .db, .sqlite, .sqlite3. Also check files with no extension by reading first 16 bytes and matching "SQLite format 3\0". Return array of `{ name, path, size }`. Skip directories named "Caches" and "tmp" for performance.
- `updateCell(dbPath, table, rowid, column, value)`: Copy-on-write strategy: copy dbPath (and .wal, .shm siblings if they exist) to a temp dir via `mkdtemp`. Open the copy with `{ readonly: false }`. Execute `UPDATE "<table>" SET "<column>" = ? WHERE rowid = ?`. Close db. Copy the modified .db file back to original path. Clean up temp. This is safe because we never write to the actively-used database.

**prefs-reader.ts** — Key-value store reading:

iOS NSUserDefaults:
- `readNSUserDefaults(dataContainer: string, bundleId: string)`: Read plist from `<dataContainer>/Library/Preferences/<bundleId>.plist`. Use `plutil -convert json -r -o - <path>` via spawn (same pattern as ios.ts plistToJson). Parse JSON output and return as `Record<string, unknown>`. Handle missing file gracefully — return empty object.
- `listPlistFiles(dataContainer: string)`: Read `<dataContainer>/Library/Preferences/` directory, return .plist filenames. Handle ENOENT gracefully — return empty array.

Android SharedPreferences:
- `readSharedPreferences(deviceId: string, packageName: string)`: Execute `adb -s <deviceId> shell run-as <packageName> ls /data/data/<packageName>/shared_prefs/`. For each .xml file, read content via `adb shell run-as <packageName> cat /data/data/<packageName>/shared_prefs/<file>`. Parse each XML file and return `Record<string, Array<{ key, value, type }>>` keyed by filename.
- `parseSharedPrefsXml(xml: string)`: Parse the well-defined SharedPreferences XML format using regex:
  - `<string name="key">value</string>` → type "string"
  - `<int name="key" value="N" />` → type "int", parse as number
  - `<boolean name="key" value="true" />` → type "boolean"
  - `<float name="key" value="N.N" />` → type "float", parse as float
  - `<long name="key" value="N" />` → type "long", parse as number
  - `<set name="key"><string>a</string>...</set>` → type "set", value as string array
  Return `Array<{ key, value, type }>`.
- Guard against `avd:` prefix device IDs. Catch "not debuggable" errors from run-as.

Use `execFile` (never `exec`) via `promisify` for all shell commands.

**routes.ts** — Fastify REST routes for database operations (follow app-management/routes.ts pattern):

Helper: `resolveContainerAndPlatform(deviceId, bundleId, fastify)` — same pattern as file-system module. Returns `{ device, adapter, containerPath, platform }`.

For Android SQLite operations: The database file must be pulled to a local temp directory first (adb pull), operated on locally with better-sqlite3, then pushed back for writes. Use a helper `withLocalDb(deviceId, packageName, dbRelativePath, readonly, fn)` that handles the pull/operate/push cycle.

For iOS SQLite operations: The database is on the host filesystem — open it directly with better-sqlite3.

Routes (all prefixed under /api/modules/database by the module system):

1. `GET /databases/:deviceId/:bundleId`: Find all SQLite databases in the app container. For iOS: call findDatabases(containerPath) directly. For Android: `adb shell run-as <pkg> find /data/data/<pkg> -name "*.db" -o -name "*.sqlite" -o -name "*.sqlite3"` (simpler than recursive walk over adb). Return `{ databases: Array<{ name, path, size }> }`.

2. `GET /tables/:deviceId/:bundleId` with query param `db` (relative path to db file): Get all tables with schema. For iOS: openReadonly the host path, call getTables. For Android: pull db to temp, openReadonly, getTables. Return `{ tables }`. Always close the database in a finally block.

3. `GET /table-data/:deviceId/:bundleId` with query params `db`, `table`, `limit` (default 50), `offset` (default 0), `orderBy?`, `orderDir?` (ASC/DESC): Get paginated table data. Open readonly, call queryTable. Return `{ rows, columns, total }` where total is the full row count from a separate COUNT(*) query.

4. `POST /query/:deviceId/:bundleId` with body `{ db: string, sql: string }`: Run arbitrary SQL. Open readonly (for SELECT queries), call runQuery. For write queries (INSERT/UPDATE/DELETE), use copy-on-write via updateCell pattern — copy to temp, open writable, execute, copy back. Return `{ type, rows?, columns?, changes? }`. Catch SqliteError and return 400 with error message.

5. `POST /update-cell/:deviceId/:bundleId` with body `{ db, table, rowid, column, value }`: Update a single cell. For iOS: call updateCell directly. For Android: pull db+wal+shm to temp, update, push back. Return `{ success: true }`.

6. `GET /prefs/:deviceId/:bundleId`: Get key-value preferences. For iOS: call readNSUserDefaults(containerPath, bundleId). For Android: call readSharedPreferences(deviceId, bundleId). Return `{ platform, prefs }` where prefs is the parsed key-value data.

Follow the CLI pattern from app-management manifest: dynamic import @simvyn/core, prefix matching, finally cleanup.
