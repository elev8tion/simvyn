**fs-store.ts** — Zustand store for file system state:
- State: `entries: FileEntry[]` (current directory listing), `currentPath: string` (relative path, default "."), `selectedDevice: string | null`, `selectedApp: string | null`, `loading: boolean`, `editingFile: { path: string, content: string } | null`, `error: string | null`.
- FileEntry type: `{ name: string, path: string, isDirectory: boolean, size: number, modified: string }`.
- Actions:
  - `fetchEntries(deviceId, bundleId, path?)`: GET `/api/modules/fs/ls/${deviceId}/${bundleId}?path=${encodeURIComponent(path)}`. Set entries from response. Update currentPath.
  - `downloadFile(deviceId, bundleId, filePath)`: Trigger browser download via `window.open(`/api/modules/fs/pull/${deviceId}/${bundleId}?path=${encodeURIComponent(filePath)}`)` or fetch as blob + create object URL + click hidden anchor.
  - `uploadFile(deviceId, bundleId, destPath, file: File)`: POST `/api/modules/fs/push/${deviceId}/${bundleId}?path=${encodeURIComponent(destPath)}` with FormData containing the file. Refresh entries after upload.
  - `openFile(deviceId, bundleId, filePath)`: GET `/api/modules/fs/read/${deviceId}/${bundleId}?path=${encodeURIComponent(filePath)}`. Set editingFile with path and content. If API returns 400 (binary file), show toast error.
  - `saveFile(deviceId, bundleId, filePath, content)`: POST `/api/modules/fs/write/${deviceId}/${bundleId}` with body `{ path: filePath, content }`. Show toast on success/error. Clear editingFile on success.
  - `closeEditor()`: Set editingFile to null.

**FileBrowser.tsx** — Directory listing with navigation:
- Breadcrumb bar showing current path segments. Each segment is clickable to navigate to that directory level. Root shows app name.
- File/directory list as a table: Icon (folder/file), Name (clickable — folders navigate, text files open editor), Size (human-readable: KB/MB), Modified (relative time or short date).
- Directory entries sorted first (folders on top), then files alphabetically.
- Action buttons per file: Download (all files), Edit (text files only — check extension), Delete (not needed for v1).
- Upload button in the header: hidden file input, triggers uploadFile on selection. Upload to current directory path.
- Empty state when no entries: "No files found" message.
- Loading spinner while fetching.

**FileEditor.tsx** — Inline text editor:
- Full-width textarea with monospace font (font-mono), dark glass-panel background.
- Header showing file path and Save/Close buttons.
- Save button calls saveFile with current textarea content.
- Close button calls closeEditor (with unsaved changes confirmation if content differs from original).
- Syntax-aware: for .json files, try to pretty-print on load via JSON.stringify(JSON.parse(content), null, 2). Show as-is if parse fails.

**FileSystemPanel.tsx** — Panel wrapper (follow AppPanel.tsx pattern):
- Device selector dropdown (same styling as other panels).
- App selector dropdown below device selector — fetch app list via `/api/modules/apps/list/${deviceId}` when device changes. Show bundleId as label.
- When both device and app selected, render FileBrowser.
- When editingFile is set in store, render FileEditor overlay/panel instead of FileBrowser.
- Register via `registerPanel("fs", FileSystemPanel)`.

Style all components with the glass-panel design system: oklch colors, backdrop-filter, border-border, text-text-primary/secondary classes. Follow the visual patterns from existing panels.

**db-store.ts** — Zustand store for database state:
- State: `databases: DbInfo[]`, `selectedDb: string | null`, `tables: TableInfo[]`, `selectedTable: string | null`, `tableData: { rows: any[], columns: any[], total: number } | null`, `queryResult: QueryResult | null`, `prefs: any | null`, `page: number`, `pageSize: number` (default 50), `orderBy: string | null`, `orderDir: "ASC" | "DESC"`, `loading: boolean`, `activeTab: "tables" | "query" | "prefs"` (default "tables").
- DbInfo: `{ name: string, path: string, size: number }`.
- TableInfo: `{ name: string, columns: Array<{name, type, notnull, pk}>, rowCount: number }`.
- QueryResult: `{ type: "rows", rows: any[], columns: any[] } | { type: "run", changes: number } | { type: "error", message: string }`.
- Actions:
  - `fetchDatabases(deviceId, bundleId)`: GET `/api/modules/database/databases/${deviceId}/${bundleId}`. Set databases.
  - `fetchTables(deviceId, bundleId, dbPath)`: GET `/api/modules/database/tables/${deviceId}/${bundleId}?db=${encodeURIComponent(dbPath)}`. Set tables, selectedDb.
  - `fetchTableData(deviceId, bundleId, dbPath, table, page?, orderBy?, orderDir?)`: GET `/api/modules/database/table-data/...` with query params. Set tableData, page, orderBy, orderDir.
  - `runQuery(deviceId, bundleId, dbPath, sql)`: POST `/api/modules/database/query/...` with body. Set queryResult. Handle errors.
  - `updateCell(deviceId, bundleId, dbPath, table, rowid, column, value)`: POST `/api/modules/database/update-cell/...`. Show toast on success. Refresh tableData.
  - `fetchPrefs(deviceId, bundleId)`: GET `/api/modules/database/prefs/${deviceId}/${bundleId}`. Set prefs.
