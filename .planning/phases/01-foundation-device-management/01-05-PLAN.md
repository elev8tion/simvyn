  <name>Task 1: Device management REST routes and WebSocket handler</name>
  <files>
    packages/modules/device-management/package.json
    packages/modules/device-management/tsconfig.json
    packages/modules/device-management/routes.ts
    packages/modules/device-management/ws-handler.ts
  </files>
  <action>
    Create packages/modules/device-management/package.json:
    - name: "@simvyn/module-device-management", type: "module"
    - dependencies: @simvyn/types: "*", @simvyn/core: "*"
    - Add to root package.json workspaces array: "packages/modules/*"

    Create routes.ts — Fastify plugin that registers device management REST endpoints:
    - GET /list → calls fastify.deviceManager.devices, returns { devices: Device[] }
    - GET /capabilities → returns per-platform capability lists from adapters
    - POST /boot → body: { deviceId: string }, calls adapter.boot(), returns { success: true }. Determine platform from device ID (look up in device list). Return 404 if device not found. Return 400 if device already booted.
    - POST /shutdown → body: { deviceId: string }, calls adapter.shutdown(). Return 404 if device not found.
    - POST /erase → body: { deviceId: string }, calls adapter.erase() (iOS only). Return 400 if Android device (erase not supported). Return 404 if device not found.
    - POST /refresh → force-refresh device list (calls deviceManager.refresh()), returns updated list

    After boot/shutdown/erase, call deviceManager.refresh() to update the device list immediately rather than waiting for the next poll tick.

    All routes should handle errors gracefully — catch adapter errors, return 500 with { error: message }.

    Create ws-handler.ts — function that registers the "devices" WebSocket channel:
    - On "subscribe": client subscribes to device updates (handled by WsBroker system channel).
    - On "list": send current device list to the requesting client.
    - On "refresh": force refresh and broadcast to all subscribers.
    - Register a listener on deviceManager's "devices-changed" event: broadcast updated device list to all "devices" channel subscribers.
    - Format: WsEnvelope with channel: "devices", type: "device-list", payload: Device[]
  </action>
  <verify>
    Start server with the modules directory pointing to packages/modules/.
    `curl http://localhost:3847/api/modules/devices/list` returns JSON array of devices.
    `curl -X POST http://localhost:3847/api/modules/devices/refresh` returns refreshed device list.
    WebSocket client subscribing to "devices" channel receives device-list updates when devices change.
  </verify>
  <done>Device REST API: list, boot, shutdown, erase, refresh, capabilities. WebSocket: real-time device list broadcasting on changes. All routes handle errors gracefully.</done>
</task>
