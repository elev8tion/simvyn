<task type="auto">
  <name>Task 2: WebSocket broker — envelope routing and connection management</name>
  <files>packages/server/src/ws-broker.ts</files>
  <action>
    Create packages/server/src/ws-broker.ts:
    - Export as a Fastify plugin (using fastify-plugin to break encapsulation — broker must be accessible to all modules).
    - On registration, create the /ws WebSocket route.

    WsBroker functionality:
    - Track connected clients: `Set<WebSocket>` with per-client subscription state (which channels each client is subscribed to).
    - On connection: add to set, send a "connected" envelope with server info.
    - On message: parse JSON as WsEnvelope, validate it has `channel` and `type` fields.
      - If channel === "system" and type === "ping" → reply with pong.
      - If channel === "system" and type === "subscribe" → add channel from payload to client's subscription set.
      - If channel === "system" and type === "unsubscribe" → remove channel from subscription set.
      - Otherwise: look up registered channel handler, dispatch if found. If not found, send error envelope back.
    - On close/error: remove from client set, clean up subscriptions.

    Channel handler registration:
    - `registerChannel(channel: string, handler: (type: string, payload: unknown, socket: WebSocket, requestId?: string) => void): void`
    - Modules call this to register their WS handlers during Fastify plugin registration.

    Broadcasting:
    - `broadcast(channel: string, type: string, payload: unknown): void` — sends envelope to all clients subscribed to that channel.
    - `send(socket: WebSocket, channel: string, type: string, payload: unknown, requestId?: string): void` — sends to specific client.
    - Always check `socket.readyState === WebSocket.OPEN` before sending.

    Decorate fastify instance with `wsBroker` so modules can access it:
    ```
    fastify.decorate("wsBroker", broker)
    ```

    Expose broker methods to Fastify type system via declaration merging.
  </action>
  <verify>
    Server starts, connect via wscat or quick Node script: `const ws = new WebSocket("ws://localhost:3847/ws"); ws.on("open", () => { ws.send(JSON.stringify({channel:"system",type:"ping",payload:{}})); });` — should receive pong envelope back.
  </verify>
  <done>WsBroker accepts connections, routes envelopes by channel, supports subscribe/unsubscribe, provides broadcast/send methods. Accessible to all modules via fastify.wsBroker decorator.</done>
</task>

<task type="auto">
  <name>Task 3: Module loader — auto-discovery and registration</name>
  <files>packages/server/src/module-loader.ts</files>
  <action>
    Create packages/server/src/module-loader.ts:
    - Export `loadModules` as a Fastify plugin.
    - Options: { modulesDir: string } — path to directory containing module folders.

    Module discovery:
    1. Read `modulesDir` with `readdir({ withFileTypes: true })`
    2. For each directory entry, look for `manifest.ts` or `manifest.js` (compiled)
    3. Dynamic import: `const mod = await import(join(modulesDir, entry.name, "manifest"))` — expect a default export conforming to SimvynModule interface
    4. Validate the manifest: must have `name`, `version`, `register` function
    5. Register the module's Fastify plugin with prefix: `/api/modules/${mod.name}`
    6. Collect module metadata (name, version, description, icon, capabilities) into a registry
    7. Log each discovered module: `fastify.log.info({ module: mod.name }, "Module loaded")`

    Module registry:
    - Decorate fastify with `moduleRegistry: Map<string, ModuleMetadata>` where ModuleMetadata = { name, version, description, icon, capabilities }
    - Expose GET /api/modules → returns array of module metadata (used by dashboard sidebar to list available modules)

    Error handling:
    - If a module's manifest fails to import or validate, log a warning and skip it (don't crash the server).
    - If modulesDir doesn't exist, log info and continue (no modules to load).

    CLI integration hook:
    - Export `getModuleCLIRegistrars(modulesDir: string): Promise<Array<{ name: string, register: (program: any) => void }>>` — used by CLI package to add module subcommands without starting the server.
  </action>

  </verify>
