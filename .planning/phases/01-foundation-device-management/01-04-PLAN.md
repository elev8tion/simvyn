    Create packages/dashboard/src/hooks/use-ws.ts:
    - Export `useWs()` hook:
      - Creates a single WebSocket connection to `/ws` (relative, uses Vite proxy in dev, same origin in prod).
      - Auto-reconnects on close with 2-second delay (max 10 retries).
      - Returns { ws: WebSocket | null, connected: boolean, send(envelope: WsEnvelope): void }
      - send() checks readyState before sending, silently drops if not connected.
      - On connect: subscribe to "devices" channel automatically (send subscribe envelope).
      - Parse incoming messages as WsEnvelope and dispatch via a Map<channel, Set<handler>> — allow components to register listeners via `useWsListener(channel, type, handler)`.
    - Export `useWsListener(channel: string, type: string, handler: (payload: unknown) => void)` — registers/unregisters on mount/unmount.
    - Use a React context provider for the WS connection (single connection for the whole app).

    Create packages/dashboard/src/stores/device-store.ts:
    - Zustand store with:
      - `devices: Device[]` — all known devices
      - `selectedDeviceId: string | null` — currently selected device
      - `broadcastMode: boolean` — target all devices
      - `setDevices(devices: Device[])` — update list
      - `selectDevice(id: string)` — set selection
      - `toggleBroadcast()` — toggle broadcast mode
      - `selectedDevice` getter — returns Device or undefined

    Create packages/dashboard/src/stores/module-store.ts:
    - Zustand store with:
      - `modules: ModuleInfo[]` — loaded from GET /api/modules
      - `activeModule: string | null` — currently visible module name
      - `setModules(modules: ModuleInfo[])` — update list
      - `setActiveModule(name: string)` — switch active module
      - `fetchModules()` — GET /api/modules and call setModules
    - ModuleInfo: { name, version, description, icon }

    Create packages/dashboard/src/components/DeviceSelector.tsx:
    - Dropdown that shows current selected device name (or "No devices" / "All devices")
    - On open: lists all devices from device-store, grouped by platform (iOS / Android)
    - Each device shows: name, state (green dot for booted, grey for shutdown), OS version
    - "All devices" toggle for broadcast mode
    - Use glass-panel styling with backdrop blur

    Create packages/dashboard/src/components/TopBar.tsx:
    - Fixed height top bar (h-14)
    - Left: "simvyn" logo text (font-medium, text-lg, accent-blue color)
    - Center/Right: DeviceSelector component
    - Right: connection status indicator (green dot when WS connected, red when disconnected)
    - Glass-panel background with bottom border

    Create packages/dashboard/src/components/Sidebar.tsx:
    - Fixed width sidebar (w-56)
    - Lists modules from module-store
    - Each item: icon (placeholder emoji or lucide icon name) + module name
    - Active module highlighted with accent-blue background tint
    - Click sets activeModule in module-store
    - Glass-panel background with right border
    - If no modules loaded, show "No modules" placeholder

    Create packages/dashboard/src/stores/panel-registry.ts:
    - A simple Map<string, React.ComponentType> that maps module names to their panel components.
    - Export `registerPanel(moduleName: string, component: React.ComponentType): void` — adds to the registry.
    - Export `getPanel(moduleName: string): React.ComponentType | undefined` — retrieves a panel.
    - Export `usePanelRegistry()` — Zustand store wrapping the registry so React re-renders when panels are registered.
    - Modules call registerPanel() at import time (side-effect import) to register their dashboard panels.

    Create packages/dashboard/src/components/ModuleShell.tsx:
    - Renders the active module's panel component looked up from panel-registry.
    - Uses `usePanelRegistry()` to get the component for `activeModule` from module-store.
    - Wraps each panel in React.Suspense with a loading skeleton (glass-panel with shimmer).
    - When no module is active, show a welcome/empty state: "Select a module from the sidebar"
    - When active module has no registered panel, show: "No panel available for {module name}"
    - Module panels are keyed by module name so state persists when switching (use `key={moduleName}` on a wrapper div that is NOT unmounted — keep panels alive with CSS display:none/block pattern or use a cache)

    Create packages/dashboard/src/App.tsx:

        - Sidebar (fixed left)
        - ModuleShell (flex-1, overflow auto)
    - On mount: fetch modules from /api/modules via module-store
    - Listen for device-list WS messages and update device-store
