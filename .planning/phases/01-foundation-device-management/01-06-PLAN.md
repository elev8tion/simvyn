<task type="auto">
  <name>Task 2: Device CLI subcommands for headless use</name>
  <files>packages/cli/src/commands/device.ts</files>
  <action>
    Create packages/cli/src/commands/device.ts:
    - Export `registerDeviceCommand(program: Command)` function
    - Register `device` subcommand group with these commands:

    `simvyn device list`:
    - Create adapters via createAvailableAdapters()
    - Create DeviceManager, call refresh()
    - Print table to stdout:
      ```
      ID                                    NAME              PLATFORM  STATE     OS VERSION
      12345678-ABCD-1234-ABCD-123456789ABC  iPhone 17 Pro     ios       booted    iOS 26.2
      avd:Pixel_6_Pro                       Pixel 6 Pro       android   shutdown  Android
      ```
    - Options: --platform <ios|android> (filter by platform), --json (output JSON instead of table)
    - No server needed — works completely headlessly.

    `simvyn device boot <id>`:
    - Create adapters, find device by ID, call adapter.boot(id)
    - Print "Booting <name>..." then poll until state changes to booted (max 60s timeout)
    - Print "✓ <name> booted" or "✗ Timed out waiting for <name> to boot"

    `simvyn device shutdown <id>`:
    - Find device, call adapter.shutdown(id)
    - Print "✓ <name> shut down"

    `simvyn device erase <id>`:
    - Find device, verify it's iOS (error if Android — erase not supported)
    - Verify device is shutdown (error if booted — must shut down first)
    - Call adapter.erase(id)
    - Print "✓ <name> erased"

    All commands should handle errors with clear messages: "Device not found: <id>", "No iOS support on this platform", etc. Exit with code 1 on error.
  </action>
  <verify>
    `node packages/cli/src/index.ts device list` prints device table (on macOS with simulators).
    `node packages/cli/src/index.ts device list --json` prints JSON array.
    `node packages/cli/src/index.ts device list --platform ios` shows only iOS devices.
    `node packages/cli/src/index.ts device boot <valid-id>` boots a simulator.
    `node packages/cli/src/index.ts device shutdown <valid-id>` shuts it down.
    `node packages/cli/src/index.ts device erase <valid-id>` erases it (after shutdown).
    `node packages/cli/src/index.ts device boot nonexistent` exits with error message.
  </verify>
  <done>Device CLI subcommands work headlessly: list (table/JSON output, platform filter), boot (with polling), shutdown, erase (iOS only). Clear error messages. No server required.</done>
