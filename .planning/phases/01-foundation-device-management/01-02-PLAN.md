  <action>
    Create packages/core/src/platform.ts:
    - Export `hasBinary(name: string): Promise<boolean>` — checks if a binary exists using `which` on macOS/Linux.
    - Export `isMacOS(): boolean` — `process.platform === "darwin"`
    - Export `isLinux(): boolean` — `process.platform === "linux"`

    Create packages/core/src/adapters/ios.ts — `createIosAdapter(): PlatformAdapter`:
    - `isAvailable()`: Try `xcrun simctl list devices --json`. Return false if fails (Linux, or Xcode not installed).
    - `listDevices()`: Parse `xcrun simctl list devices --json`. Iterate ALL runtimes. Filter `isAvailable === true`. Parse runtime key for OS version: `com.apple.CoreSimulator.SimRuntime.iOS-26-2` → `iOS 26.2`. Map `state` ("Booted"→"booted", "Shutdown"→"shutdown", "Creating"→"creating", "ShuttingDown"→"shutting-down"). Include ALL available devices (not just booted — unlike sim-location which filters to booted only). Set `deviceType` from `deviceTypeIdentifier` (split on "." take last segment, replace hyphens with spaces).
    - `boot(id)`: `xcrun simctl boot <id>`. Swallow "already booted" error.
    - `shutdown(id)`: `xcrun simctl shutdown <id>`. Swallow "already shutdown" error.
    - `erase(id)`: `xcrun simctl erase <id>`.
    - `capabilities()`: Return full iOS capability set: ["setLocation", "push", "screenshot", "screenRecord", "erase", "statusBar", "privacy", "ui", "clipboard", "addMedia", "logs", "deepLinks", "appManagement"]

    Use `node:child_process.execFile` promisified for all commands. Never use `exec` (shell injection risk).

    Create packages/core/src/adapters/android.ts — `createAndroidAdapter(): PlatformAdapter`:
    - `isAvailable()`: Try `adb version`. Return false if fails.
    - `listDevices()`:
      1. Get available AVDs via `emulator -list-avds` (one per line). If `emulator` command fails, skip AVD detection (still detect USB devices).
      2. Get running devices via `adb devices` (skip first "List of devices attached" line, split tab-separated, filter status "device" or "emulator").
      3. For running emulators (serial starts with "emulator-"), resolve AVD name via `adb -s <serial> emu avd name` (take first line, trim).
      4. For USB physical devices (serial doesn't start with "emulator-"), get model via `adb -s <serial> shell getprop ro.product.model` and Android version via `adb -s <serial> shell getprop ro.build.version.release`.
      5. Merge: running emulators (state: booted) + USB devices (state: booted) + available AVDs not currently booted (state: shutdown, id: `avd:<name>`).
    - `boot(id)`: If id starts with "avd:", extract AVD name, spawn `emulator @<name>` detached (stdio: "ignore", unref). Then poll `adb devices` up to 60 seconds waiting for a new emulator serial to appear with status "device".
    - `shutdown(id)`: `adb -s <id> emu kill`. For USB devices, this is a no-op (can't kill a physical device).
    - `erase`: undefined (Android doesn't support erase via adb).
    - `capabilities()`: ["setLocation", "screenshot", "screenRecord", "logs", "deepLinks", "appManagement", "addMedia"]

    Create packages/core/src/adapters/index.ts:
    - Export createIosAdapter, createAndroidAdapter
    - Export `createAvailableAdapters(): Promise<PlatformAdapter[]>` — creates both, filters to available ones. On Linux, iOS adapter won't be available.

    Wrap all execFile calls in try/catch. Log errors via console.warn (will be replaced with pino in server plan). Never throw for expected failures (binary not found, device already booted).
